# 跳转模式

本规格描述 MoeURL 跳转模式的完整目标形态。具体实现顺序以 [v0.0.1 范围](../product/scope-v0.0.1.md) 和 [路线图](../product/roadmap.md) 为准。

## 直接跳转

访问短链后直接进入目标链接。

适合：

- 工具类短链。
- 内部分享。
- 对速度要求高的场景。

页面效果：

- 用户通常看不到中间页面。
- 系统仍可记录访问事件。

v0.0.1 只要求实现直接跳转。

## 中间页跳转

访问短链后先展示中间页，再自动跳转。

中间页可包含：

- 目标域名。
- 倒计时。
- 立即跳转按钮。
- 安全提示。
- 背景图。
- 自定义文案。
- 页脚。

适合：

- 外链提醒。
- 广告展示。
- 品牌曝光。
- 需要缓冲提示的场景。

中间页跳转计划在 v0.1.0 实现。

## 确认页跳转

访问短链后需要用户主动确认。

确认页可包含：

- 目标链接或目标域名。
- 风险提示。
- 继续访问按钮。
- 返回按钮。

适合：

- 外部未知链接。
- 风险链接提示。
- 需要访问者主动确认后再继续访问的链接。
- 管理员要求手动确认的链接。

确认页跳转属于后续版本能力。

## 访问密码

访问短链后要求输入密码，验证通过后再根据短链配置进入直接跳转、中间页或确认页。

访问密码是否可用由用户组权限控制。

访问密码只保存哈希值，不保存明文。管理员和创建者均不能查看原密码，只能重置密码。密码验证失败不应计入有效访问量，但可以记录为访问条件失败事件，用于后续风控、审计或统计分析。

访问密码计划在 v0.1.0 实现。

## 跳转页模板

系统可以内置多种跳转页模板：

- 默认中间页。
- 极简确认页。
- 图片背景页。
- 错误提示页。

用户能否使用某个模板，由用户组权限控制。

每个模板可以声明自己的元数据字段。用户选择某种模板后，创建短链表单会出现该模板对应的填写项。

模板元数据设计原则：

- 由模板定义字段，不允许用户随意注入任意页面结构。
- 字段应有类型、默认值、校验规则和权限控制。
- 管理员可以配置模板默认值和适用用户组。
- 普通用户只能在创建或编辑指定短链时填写自己可用的字段。
