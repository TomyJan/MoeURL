# 主题与多语言

## 多语言设计

系统初版建议支持：

- 简体中文。
- 英语。

语言切换器中可以显示为「简体中文 / English」。

后续可扩展：

- 繁体中文。
- 日语。

多语言覆盖范围：

- 首页。
- 登录页。
- 用户中心。
- 我的短链页。
- 统计页。
- 管理后台。
- 跳转页。
- 错误页。
- 系统提示。

语言选择优先级：

```text
用户手动选择
  ↓
用户账号偏好
  ↓
浏览器语言
  ↓
系统默认语言
```

v0.0.1 应实现多语言基础架构，至少支持简体中文和英语。

## 主题模式

系统支持 3 种主题模式：

- 跟随系统。
- 浅色模式。
- 深色模式。

未登录访问者默认跟随系统。

登录用户可以在个人设置中保存偏好。

v0.0.1 应实现主题模式基础架构，至少支持跟随系统、浅色模式和深色模式。

v0.0.1 前端初始化时即应接入 MoeURL 自定义主题。默认主题采用 Material Design 3 风格，可以先写死在前端配置中，不要求提供后台主题编辑能力。

## PWA 基础能力

MoeURL 从 v0.0.1 开始支持 PWA 基础能力。

v0.0.1 应包含：

- Web App Manifest。
- 应用图标和启动主题色。
- 基础 Service Worker 注册。
- App Shell 和静态资源缓存。
- 移动端安装后的基础启动体验。

v0.0.1 不缓存登录态 API、短链业务数据和权限相关响应，避免权限状态和业务数据陈旧。

## 响应式布局

所有核心页面都应适配：

- 手机。
- 平板。
- 桌面。

短链访问页尤其需要优先保证移动端体验。

v0.0.1 应实现响应式基础布局。

## 站点品牌

管理员可以配置：

- 站点名称。
- Logo。
- Favicon。
- 主题色。
- 首页标语。
- 默认背景图。

站点品牌后台配置可以后续逐步完善。

## 页脚配置

页脚是系统主题配置的一部分，只允许管理员配置。

可配置内容包括：

- 版权信息。
- 备案号。
- 自定义链接。
- Powered by 文案（可配置是否展示）。
- 简单自定义文本。

页脚配置应用于首页、系统页面、跳转页和错误页。具体页面是否展示某些页脚内容，可以由主题策略控制。

系统主题和页脚后台配置不属于 v0.0.1 范围。

## 跳转页模板外观

系统可以内置多种跳转页模板：

- 默认中间页。
- 极简确认页。
- 图片背景页。
- 错误提示页。

每个模板可以声明自己的元数据字段。用户选择某种模板后，创建短链表单会出现该模板对应的填写项。

示例：

| 模板 | 可填写元数据 |
| --- | --- |
| 默认中间页 | 提示文案、倒计时、按钮文案。 |
| 图片背景页 | 背景图、主标题、副标题、按钮文案。 |
| 极简确认页 | 风险提示、确认按钮文案、返回按钮文案。 |

模板元数据设计原则：

- 由模板定义字段，不允许用户随意注入任意页面结构。
- 字段应有类型、默认值、校验规则和权限控制。
- 管理员可以配置模板默认值和适用用户组。
- 普通用户只能在创建或编辑指定短链时填写自己可用的字段。

跳转页模板和模板元数据结构在 v0.0.1 只要求预留，不要求实现完整管理界面。
