# 短链

## 短链访问形式

MoeURL 使用简洁的路径型短链访问形式，例如：

```text
https://xxx.com/abcd
```

其中，`xxx.com` 是系统中已配置并启用的短链访问域名，`abcd` 是短码。访问者打开该地址后，系统根据访问域名和短码识别对应短链，并按短链配置执行跳转、展示中间页、展示确认页或要求输入访问密码。

短码在全系统范围内全局唯一，不随短链访问域名变化而重复。也就是说，同一个短码不能在不同短链访问域名下重复使用。

系统页面路径和短链短码路径需要有明确优先级：系统固定路由优先，短链短码路由其次，最后才进入 404 状态页。系统应在短码创建流程中预留保留路径校验入口。v0.0.1 可以先覆盖当前已知固定路由，完整保留路径策略后续继续完善。

## 未登录访问首页

未登录用户进入首页时，系统以 `guest` 身份判断可见内容和可用功能。

首页的短链创建区域作为核心组件应保持展示。若 `guest` 拥有创建短链权限，则表单可直接使用；若 `guest` 没有创建短链权限，则表单置灰，并展示清晰提示和登录入口。

## 创建短链流程

创建短链功能默认放在首页。用户也可以从其他页面通过快捷入口回到首页进行创建。

系统根据权限动态控制可用字段。

基础流程：

```text
输入目标链接
  ↓
选择可用域名
  ↓
选择短码生成方式
  ↓
选择跳转方式
  ↓
选择跳转页模板（如适用）
  ↓
填写模板元数据（如适用）
  ↓
配置可用高级选项
  ↓
生成短链
  ↓
复制或继续管理
```

页面需要做到：

- 对常见核心字段，无权限时优先置灰并说明原因。
- 对高阶或不相关字段，可根据权限隐藏。
- 常用字段默认展开。
- 高级设置默认折叠。
- 创建成功后提供复制、打开、继续创建和返回我的短链列表入口。v0.0.1 不实现独立短链详情页。

## 目标链接安全校验

目标链接在创建和编辑时必须通过基础安全校验。

v0.0.1 至少应要求目标链接是合法 URL，并默认只允许 `http://` 和 `https://` 协议；禁止 `javascript:`、`data:`、`file:` 等不适合作为跳转目标的协议。

系统还应预留目标链接审核状态和拦截点，便于后续加入黑名单、白名单、外部安全数据源、人工复审和风险链接强制确认页。

v0.0.1 默认不开放 `guest` 创建短链。若后续开放公开创建能力，需要增加更严格的目标地址风险校验，包括内网地址、localhost、私有 IP 段、当前系统管理路径、控制字符、不可见字符和 URL 最大长度等边界。

系统应预留避免明显短链循环的校验入口，例如短链直接指向自身。v0.0.1 不要求完整实现跨短链循环检测；是否允许短链指向其他短链，可以由后续风控策略进一步限制。

## 访问短链流程

访问者打开短链后，系统按以下顺序处理：

```text
识别域名和短码
  ↓
查找短链
  ↓
预留 `short_link_opened` 事件记录点
  ↓
检查短链状态
  ↓
检查访问条件
  ↓
根据检查结果调用事件记录接口记录 `redirect_blocked` 或继续处理
  ↓
按跳转方式展示或跳转
  ↓
调用事件记录接口预留 `redirect_initiated` / `redirect_response_sent` 等事件
```

可能结果包括：

- 直接跳转到目标链接。
- 展示中间页后自动跳转。
- 展示确认页，等待用户点击后跳转。
- 要求输入访问密码。
- 显示短链不存在页面。
- 显示短链已禁用页面。
- 显示短链已过期页面。
- 显示访问次数已达上限页面。

所有访问页都应支持多语言、自动深色模式、响应式布局，并使用系统主题和页脚配置。

## 短链状态模型

系统应为短链定义统一状态，避免访问流程、管理后台和统计口径各自判断。

v0.0.1 必须实现的最小状态包括：

| 状态 | 说明 |
| --- | --- |
| `active` | 启用中，可以按配置访问。 |
| `disabled` | 已禁用，不允许访问目标链接。 |

软删除通过 `deleted_at` 表达，不作为普通业务状态复用。删除后普通列表默认不展示，访问时按不存在处理。

后续版本可以扩展以下状态：

| 状态 | 说明 |
| --- | --- |
| `expired` | 已过期，不允许访问目标链接。 |
| `exhausted` | 访问次数已达上限，不允许访问目标链接。 |
| `pending` | 待审核，预留给后续风控和人工审核。 |
| `rejected` | 审核拒绝，预留给后续风控和人工审核。 |

访问短链时，应先识别短链，再根据状态决定继续检查访问条件、展示状态页或返回 404。

## 删除策略

删除策略采用软删除，便于审计，并防止短码被复用造成误导。

v0.0.1 删除短链采用软删除。删除后普通列表默认不展示，访问时按不存在处理，短码永久保留占用，不进入自动复用池。v0.0.1 不提供短码释放、恢复或复用能力。
