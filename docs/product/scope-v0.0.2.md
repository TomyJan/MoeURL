# v0.0.2 范围

## 目标

v0.0.2 的目标是在 v0.0.1 基础可用闭环之上，完成发布后第一轮质量补强。

本版本不追求进入 v0.1.0 的完整 MVP 增强能力，而是优先处理以下问题：

- 让 v0.0.1 的自动化和手工验收记录闭环。
- 补齐管理员用户管理的最小可用能力。
- 改善管理员短链管理和列表筛选体验。
- 扩展 E2E、覆盖率和部署验证，降低回归风险。
- 完善自托管部署说明和运行前检查。

v0.0.2 仍应保持小步版本特征。统计、中间页跳转、访问密码、二维码和短链过期时间等能力应进入后续版本规划，不放入本版本必须实现范围。

## 必须实现

- 更新 v0.0.1 验收记录，明确自动化验证、Docker 验证和剩余手工验证状态。
- 新增 v0.0.2 实施计划、任务清单和验收清单。
- 管理员用户列表最小入口。
- 管理员查看用户的账号、昵称、用户组、状态和创建时间。
- 管理员启用或禁用普通用户。
- 管理员修改普通用户昵称。
- 管理员重置普通用户密码。
- 管理员用户管理操作必须受 `admin:access` 权限保护。
- 禁止通过管理入口禁用、修改或重置内置 `guest` 用户。
- 禁止普通用户访问用户管理入口。
- 管理员短链列表支持按状态筛选。
- 管理员短链列表支持按短码、目标 URL 或所属用户名关键词搜索。
- 我的短链列表支持按状态筛选。
- 管理员用户管理、短链筛选和权限失败场景纳入 E2E 覆盖。
- 前端覆盖率门禁继续保持 Statements、Branches、Functions、Lines 均为 100%。
- 后端业务源码覆盖率门禁继续保持 100%。
- README 补充 v0.0.2 质量门禁和部署验证说明。
- Docker Compose 启动、健康检查和初始化入口应有可复现验证步骤。

## 应该预留

- 用户详情页入口，但 v0.0.2 不要求完整详情页。
- 用户删除、批量导入和批量操作的服务边界，但不实现 UI。
- 用户组权限可视化管理的数据边界，但不实现权限编辑。
- 管理员短链列表的分页、排序和筛选参数扩展边界。
- 操作记录或审计日志调用点，但不要求事件入库或审计查询页。

## 暂不实现

- 用户自助注册。
- OIDC 登录。
- 禁用本地登录。
- 完整用户中心仪表盘。
- 完整用户详情页。
- 用户删除和批量导入。
- 多用户组和用户组权限可视化配置。
- 权限预设模型的完整可视化配置。
- 完整域名管理页面。
- 中间页跳转。
- 确认页跳转。
- 访问密码。
- 短链过期时间。
- 最大访问次数。
- 二维码能力。
- 统计页、统计聚合和访问明细查询。
- 审计日志查询页面。
- 自动审核和外部安全数据源接入。

## 完成标准

- v0.0.2 文档入口能清晰说明当前版本范围和实施顺序。
- 管理员可以进入用户列表。
- 管理员可以禁用、启用、修改昵称和重置普通用户密码。
- 普通用户不能进入或调用用户管理能力。
- 内置 `guest` 用户不能通过管理入口被修改。
- 被禁用用户不能登录，也不能继续执行需要授权的操作。
- 管理员可以按状态和关键词筛选全站短链。
- 用户可以按状态筛选自己的短链。
- 新增能力有后端测试、前端测试和 E2E 覆盖。
- 后端和前端覆盖率门禁保持 100%。
- `go test ./...`、`go vet ./...`、`pnpm lint`、`pnpm typecheck`、`pnpm test`、`pnpm test:coverage`、`pnpm build` 和 E2E 测试通过。
- Docker Compose 环境可以完成健康检查和首次初始化入口验证。
