# v0.0.3 范围

## 目标

v0.0.3 的目标是在 v0.0.2 质量补强闭环之后，完成第一轮 UI/UX 精细化重设计。

本版本不是 v0.1.0 的业务增强版本，不追求新增统计、访问密码、二维码或跳转模式。v0.0.3 的主线是：

```text
主题重建 → 首页重设计 → 短链生成组件复用 → 控制台框架统一 → 既有页面体验迁移
```

v0.0.3 应让 MoeURL 从「功能可用」推进到「产品形态清晰、视觉稳定、后续页面可持续扩展」。本阶段所有实现都应围绕既有 v0.0.2 功能展开，不改变后端核心业务契约。

## 必须实现

- 新增 v0.0.3 产品范围、实施计划、任务清单和验收清单。
- 建立新的浅色主题，采用「清透产品感」方向。
- 建立新的深色主题，采用「深墨绿清透」方向。
- 将主题 token 接入 Vuetify 3 主题系统，保留自动深色模式和手动切换能力。
- 提升全局圆角系统，主要容器、面板、弹窗和按钮应体现大圆角风格。
- 首页顶部导航视觉融入背景，不做独立背景的传统导航栏。
- 首页左侧展示名称和 Logo，右侧根据登录态展示「登录」或「头像 + 昵称」。
- 首页已登录头像昵称入口点击进入控制台。
- 首页首屏占满主要视口，并以短链快速生成为视觉中心。
- 首页首屏下方允许继续滚动，展示轻量产品介绍。
- 抽象可复用短链生成组件，复用现有创建短链能力和权限处理逻辑。
- 首页使用短链生成组件的 inline 形态。
- 控制台通过「新建短链」按钮打开短链生成浮窗。
- 控制台移动端短链生成浮窗使用底部抽屉形态。
- 控制台桌面端采用左侧导航 + 主内容区布局。
- 控制台移动端采用顶部栏 + 汉堡菜单 + 抽屉导航布局。
- 桌面控制台账号入口放在左侧导航底部，不放在主内容区右上角。
- 移动控制台顶部右侧保留头像入口。
- 控制台主内容区右上角只承载页面级动作。
- 我的短链、全站短链、用户管理和创建用户页面迁移到统一控制台框架。
- 管理入口继续根据权限展示，普通用户不得看到不可访问的管理页面入口。
- 空状态、错误状态、权限不足状态和加载状态符合新视觉系统。
- 前端测试覆盖主题、首页、短链生成组件、控制台导航和关键页面迁移。
- 现有 v0.0.2 功能、权限行为、E2E 主流程和覆盖率门禁不回退。

- 后续系统设置导航信息架构，但 v0.0.3 不展示未实现入口。

## 设计基线

v0.0.3 设计以 [v0.0.3 UI/UX 精细化重设计](../superpowers/specs/2026-06-08-v0.0.3-ui-ux-redesign.md) 为主要依据。

执行时应特别遵守以下决策：

- 浅色主题采用清透、轻量、低压迫感的产品气质。
- 深色主题采用低眩光深墨绿基调，不使用高饱和紫蓝或纯黑后台风格。
- 首页不是营销落地页，第一任务是快速生成短链。
- 控制台是工作台，不使用夸张英雄区或装饰性大卡片堆叠。
- 控制台导航只展示当前已可用入口，不用禁用菜单项提前占位。
- 短链生成组件必须复用，避免首页和控制台出现两套创建逻辑。
- UI 精细化不得破坏 v0.0.2 已完成的权限、筛选、搜索和用户管理能力。

## 完成标准

- 文档入口能清晰说明 v0.0.3 的目标、范围、实施顺序和验收标准。
- 首页桌面端和移动端都以短链快速生成为视觉中心。
- 首页导航没有独立背景，品牌、登录入口和账号入口清晰。
- 已登录用户可从首页账号入口进入控制台。
- 首页可继续下翻查看轻量介绍区。
- 浅色主题符合「清透产品感」方向。
- 深色主题符合「深墨绿清透」方向。
- 主要页面、面板、弹窗、按钮和输入框使用新的大圆角规则。
- 首页和控制台复用同一套短链生成组件和创建逻辑。
- 控制台桌面端使用左侧导航，账号入口位于左侧底部。
- 控制台移动端使用汉堡菜单和抽屉导航。
- 我的短链、全站短链、用户管理和创建用户页面已进入统一控制台框架。
- 未实现的统计、系统设置、二维码、访问密码和过期时间不出现在可点击主流程中。
- 普通用户、管理员和未登录访问者的可见入口符合权限边界。
- `cd web && pnpm lint`、`pnpm typecheck`、`pnpm test`、`pnpm test:coverage`、`pnpm build` 通过。
- 关键 E2E 主流程通过，包括首页创建短链、登录进入控制台、控制台创建短链和管理员页面访问。
- `go test ./...` 和 `go vet ./...` 通过，确认前端重构没有引入后端回归。
