# v0.1.0 范围

## 目标

v0.1.0 完成短链统计分析闭环：在 v0.0.4 的成功跳转事件与列表基础统计之上，提供按短链查看的近 7 天趋势、来源、设备和地区聚合。

主线为：

```text
最小化事件维度采集 → SQL 聚合 → 权限 API → 控制台分析页 → E2E 验收
```

## 必须实现

- `short_link_event` 保存来源主机名、设备类型和可选国家代码，不保存原始 IP、完整 User-Agent 或完整 Referer。
- 成功写出跳转响应后，沿用有界异步 recorder 尽力写入事件；失败不得影响跳转。
- 我的短链与管理员短链统计 API 返回概览、连续 7 天趋势及维度聚合。
- 普通用户只能查看自己的短链，管理员可查看全站短链。
- `/analytics` 从占位页变为真实页面，支持从两个短链列表进入。
- 统计页展示总访问量、今日访问量、最近访问时间、趋势、来源、设备和地区。
- CI 运行统计主流程 E2E，且失败时提供 Playwright 诊断产物。

## 数据与权限边界

- 访问量只统计 `redirect_response_sent`。
- 仅保存 Referer 主机名和归类后的设备类型。
- `MOEURL_ANALYTICS_COUNTRY_HEADER` 未配置时不采集国家代码；配置该请求头的部署者必须由可信反向代理覆盖或移除客户端同名请求头。
- 空维度显示为「未知」，最多返回 10 个实际维度值，其余合并为「其他」。
- 趋势按数据库当前日期返回固定连续 7 天，空日期访问量为 `0`。

## 暂不实现

- 访问明细、导出、任意日期范围和近 30 天趋势。
- 浏览器、操作系统和 GeoIP 数据库。
- 访问密码、过期时间、二维码、中间页、确认页、用户中心和权限预设。

## 完成标准

- 统计 API、控制台分析页和列表入口通过自动化测试。
- 迁移、SQLC 查询、权限边界和匿名化字段均有后端测试。
- `go test ./...`、`go vet ./...`、后端 100% 覆盖率门禁，以及前端 lint、类型检查、测试、覆盖率、构建和 E2E 全部通过。
