# 路线图

MoeURL 按「基础可用闭环 → 发布质量补强 → 产品体验精细化 → MVP 增强闭环 → 增强能力」逐步推进。

## v0.0.1 基础可用闭环

v0.0.1 聚焦初始化、本地登录、创建短链、访问短链、基础权限和基础界面能力。

详细范围见 [v0.0.1 范围](scope-v0.0.1.md)。

## v0.0.2 发布后质量补强闭环

v0.0.2 聚焦 v0.0.1 合并后的发布质量、管理体验和验证闭环。

详细范围见 [v0.0.2 范围](scope-v0.0.2.md)。

v0.0.2 建议补充：

- v0.0.1 验收记录闭环。
- 管理员用户列表。
- 管理员启用、禁用、修改昵称和重置普通用户密码。
- 我的短链状态筛选。
- 管理员全站短链状态筛选和关键词搜索。
- E2E、覆盖率和 Docker Compose 验证补强。
- README 和部署验证说明补强。

## v0.0.3 UI/UX 精细化重设计

v0.0.3 聚焦 MoeURL 的第一轮产品体验精细化，让已有功能从「能用」推进到「体验稳定、视觉完整、后续页面可扩展」。

详细范围见 [v0.0.3 范围](scope-v0.0.3.md)。

v0.0.3 建议补充：

- 新浅色主题，采用「雾蓝石墨」方向下的冷白、雾蓝和石墨文字。
- 新深色主题，采用「雾蓝石墨」方向下的蓝黑、钢蓝表面和冰蓝主操作。
- 大圆角视觉系统和全局表面规则。
- 首页融合式顶部导航。
- 首页首屏短链快速生成。
- 首页轻量下翻介绍区。
- 可复用短链生成组件。
- 控制台「新建短链」浮窗。
- 桌面端控制台左侧导航。
- 移动端控制台汉堡菜单和抽屉导航。
- 控制台账号入口策略。
- 我的短链、全站短链、用户管理和创建用户页面迁移到统一控制台框架。

v0.0.3 不补充统计、访问密码、短链过期时间、中间页跳转、确认页跳转和二维码能力。

## v0.0.4 基础统计前置

v0.0.4 聚焦短链访问基础统计，让已有短链列表可以展示访问表现，并为 v0.1.0 的统计页打下数据基础。

详细范围见 [v0.0.4 范围](scope-v0.0.4.md)。

v0.0.4 建议补充：

- 短链成功跳转事件持久化。
- 我的短链列表展示总访问量、今日访问量和最近访问时间。
- 管理员全站短链列表展示总访问量、今日访问量和最近访问时间。
- 统计查询和 API 响应测试。
- 控制台列表统计展示测试。

v0.0.4 不补充独立统计分析页、短链详情页、近 7 天趋势图、来源/设备/地区维度、访问明细、统计导出、访问密码、短链过期时间、中间页跳转、确认页跳转和二维码能力。

## v0.1.0 MVP 增强闭环

v0.1.0 建议补充：

- 用户中心。
- 基础统计。
- 基础维度统计，例如来源、设备、地区。
- 中间页跳转。
- 访问密码。
- 短链过期时间。
- 二维码基础能力。
- 权限预设模型。
- 多语言和主题模式的体验完善。
- 系统主题和页脚配置。

v0.1.0 统计最低完成标准：

- 每条短链展示总访问量。
- 每条短链展示今日访问量。
- 每条短链展示近 7 天访问趋势。
- 支持按来源、设备类型、地区查看聚合统计。
- 不要求展示单次访问明细。
- 不要求导出统计数据。

## 非目标

v0.0.1 不包含以下能力：

- 公开 SaaS 多租户计费。
- 团队空间或组织空间。
- 用户自定义域名绑定。
- 高级风控审核。
- Webhook。
- 批量导入和批量导出。
- 自定义 CSS。
- 高级模板编辑。
- 完整统计分析平台。

这些能力可以在后续版本中按实际需求单独规划，不应阻塞 v0.0.1 的基础可用闭环。
