# 路线图

MoeURL 按「基础可用闭环 → MVP 增强闭环 → 增强能力」逐步推进。

## v0.0.1 基础可用闭环

v0.0.1 聚焦初始化、本地登录、创建短链、访问短链、基础权限和基础界面能力。

详细范围见 [v0.0.1 范围](scope-v0.0.1.md)。

## v0.1.0 MVP 增强闭环

v0.1.0 建议补充：

- 用户中心。
- 基础统计。
- 基础维度统计，例如来源、设备、地区。
- 中间页跳转。
- 访问密码。
- 短链过期时间。
- 二维码基础能力。
- 权限预设模型。
- 多语言和主题模式的体验完善。
- 系统主题和页脚配置。

v0.1.0 统计最低完成标准：

- 每条短链展示总访问量。
- 每条短链展示今日访问量。
- 每条短链展示近 7 天访问趋势。
- 支持按来源、设备类型、地区查看聚合统计。
- 不要求展示单次访问明细。
- 不要求导出统计数据。

- 操作记录。
- 更多跳转页模板。
- 自定义 Open Graph 信息。
- 批量导入。
- 批量导出。
- Webhook。
- 更细粒度权限。
- 自定义 CSS。
- 高级模板编辑。
- 自动审核。

## 非目标

v0.0.1 不包含以下能力：

- 公开 SaaS 多租户计费。
- 团队空间或组织空间。
- 用户自定义域名绑定。
- 高级风控审核。
- Webhook。
- 批量导入和批量导出。
- 自定义 CSS。
- 高级模板编辑。
- 完整统计分析平台。

这些能力可以在后续版本中按实际需求单独规划，不应阻塞 v0.0.1 的基础可用闭环。
