# v0.0.4 范围

## 目标

v0.0.4 的目标是在 v0.0.3 UI/UX 精细化完成后，补齐短链访问的基础统计前置能力。

本版本不是完整统计分析版本，不实现来源、设备、地区、访问明细、导出和趋势图表。v0.0.4 的主线是：

```text
访问事件入库 → 列表基础统计聚合 → 控制台基础统计展示 → 验证闭环
```

v0.0.4 应让用户和管理员在既有短链列表中看到短链的基础访问表现，为 v0.1.0 的统计页和更完整维度统计打下稳定数据基础。

## 必须实现

- 新增 v0.0.4 产品范围、实施计划、任务清单和验收清单。
- 新增短链访问事件持久化表，记录已成功写出跳转响应的事件。
- 通过 Goose migration 管理事件表结构。
- 使用 SQLC 查询短链基础统计数据。
- HTTP handler 成功写出跳转响应后记录 `redirect_response_sent` 事件。
- 短链不存在、软删除或禁用时不计入访问量。
- 我的短链列表返回基础统计字段：
  - 总访问量。
  - 今日访问量。
  - 最近访问时间。
- 管理员全站短链列表返回同样的基础统计字段。
- 控制台短链列表展示总访问量、今日访问量和最近访问时间。
- 空统计状态应显示为可读占位，例如 `0` 或「暂无访问」。
- 现有短链创建、访问、启用、禁用、删除、筛选和搜索能力不回退。
- 自动化测试覆盖事件记录、统计聚合、API 响应和前端展示。

## 应该预留

- 事件表保留 `event_type` 字段，便于后续区分更多访问事件。
- 事件表保留 `created_at` 字段，便于后续按时间窗口统计。
- 统计查询应能继续扩展到近 7 天和近 30 天趋势。
- 前端短链模型应保留基础统计对象边界，便于后续统计页复用。

## 暂不实现

- 独立统计分析页。
- 短链详情页。
- 近 7 天趋势图。
- 近 30 天趋势图。
- 来源页面统计。
- 浏览器、操作系统和设备类型统计。
- 地区统计。
- 最近访问明细。
- 统计数据导出。
- 访问 IP、User-Agent 和 Referer 的持久化。
- 访问密码、短链过期时间、二维码、中间页跳转和确认页跳转。

## 设计基线

v0.0.4 以 [统计与事件](../specs/statistics-and-events.md)、[短链](../specs/short-links.md)、[导航与页面](../specs/navigation-and-pages.md) 和 [权限模型](../specs/permissions.md) 为产品依据。

执行时应特别遵守以下决策：

- 统计口径只统计已成功写出跳转响应的访问。
- 不存在、软删除、禁用和后续被访问条件拦截的短链不计入访问量。
- v0.0.4 只展示基础统计，不开放完整统计页和详细明细。
- 统计数据跟随既有短链列表权限边界：普通用户只能看到自己的短链统计，管理员可以在全站短链列表看到全站短链统计。
- 事件记录失败不得阻断短链跳转，但应通过测试确认正常路径会写入事件。

## 完成标准

- 文档入口能清晰说明 v0.0.4 的目标、范围、实施顺序和验收标准。
- 数据库迁移可以创建和回滚短链访问事件表。
- 成功写出短链跳转响应后，事件表写入 `redirect_response_sent` 事件。
- 短链列表 API 返回总访问量、今日访问量和最近访问时间。
- 控制台我的短链和全站短链列表展示基础统计。
- 统计字段在无访问时显示稳定，不造成前端报错或布局溢出。
- `go test ./...` 和 `go vet ./...` 通过。
- 后端覆盖率门禁保持通过。
- `cd web && pnpm lint`、`pnpm typecheck`、`pnpm test`、`pnpm test:coverage` 和 `pnpm build` 通过。
- 关键 E2E 主流程不回退。
