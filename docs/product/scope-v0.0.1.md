# v0.0.1 范围

## 目标

v0.0.1 的目标是让系统完成从初始化、本地登录、创建短链到访问短链的基础可用闭环，并建立后续演进所需的基础架构。

v0.0.1 不追求完整后台和完整权限管理界面。凡是会显著拉高第一版复杂度、但不影响基础功能闭环的能力，都应优先作为模型预留或后续版本实现。

这里的「后续实现」不代表初版可以写成临时逻辑。初版实现时仍应按长期架构预留必要抽象，只是暂不开放完整功能或完整管理界面。

## 必须实现

- 首次初始化流程。
- 本地登录流程。
- 本地用户系统的最小模型。
- 管理员创建普通用户的最小入口。
- 内置 `guest` 用户和 `guest` 用户组。
- 内置 `user` 用户组和 `admin` 用户组。
- 初始化时写入固定内置权限模型，不提供权限模型选择器。
- 一个用户只启用一个主用户组。
- 最小权限判断。
- 首页。
- 首页创建短链。
- 目标链接基础安全校验。
- 直接跳转。
- 我的短链列表。
- 复制短链、打开短链、启用或禁用短链、软删除短链。
- 短链最小状态模型：`active`、`disabled` 和软删除，其中软删除通过 `deleted_at` 表达。
- 管理员查看和管理全站短链的最小入口。
- 单全局默认短链访问域名。
- 系统固定路由与短链短码路由的优先级处理。
- 保留路径校验入口，先覆盖当前已知固定路由。
- 响应式基础布局。
- 多语言基础架构，至少支持简体中文和英语。
- 主题模式基础架构，至少支持跟随系统、浅色模式和深色模式。

## 最小权限集合

v0.0.1 只要求实现支撑基础闭环的权限项。其他权限可以先在模型设计上预留，不要求提供完整可视化配置。

v0.0.1 必须包含的最小权限项包括：

- `short_link:create`：创建短链。
- `short_link:read_own`：查看自己的短链。
- `short_link:update_own`：编辑、启用或禁用自己的短链。
- `short_link:delete_own`：删除自己的短链。
- `short_link:read_all`：查看全站短链。
- `short_link:update_all`：更新全站短链。
- `short_link:delete_all`：删除全站短链。
- `domain:use_default`：使用默认短链访问域名。
- `admin:access`：访问基础管理入口。

`guest` 用户组默认不授予上述实际操作权限。`user` 用户组默认可拥有自己的短链创建和管理权限。`admin` 用户组默认拥有系统最高管理权限。

## 应该预留

- 登录方式扩展入口。
- OIDC 提供商模型或配置扩展点。
- 用户组权限管理的扩展结构。
- 多域名模型和域名权限扩展结构。
- 统计事件类型、统一事件记录接口和访问流程调用点；v0.0.1 不要求事件入库、统计展示、统计聚合或访问明细查询。
- 扩展短链状态：`expired`、`exhausted`、`pending`、`rejected`。
- 审核状态字段和风控拦截点。
- 跳转页模板和模板元数据结构。
- 外部接口鉴权扩展入口。
- 操作记录扩展点。

## 暂不实现

- 用户自助注册。
- OIDC 登录。
- 禁用本地登录。
- 中间页跳转。
- 确认页跳转。
- 访问密码。
- 二维码能力。
- 完整用户中心仪表盘。
- 独立短链详情页和短链详情完整页。
- 事件入库、统计页、统计聚合和访问明细查询。
- 完整域名管理页面。
- 用户默认域名和用户组默认域名。
- 用户组权限的完整可视化配置。
- 权限模型选择器和权限预设模型的完整可视化配置。
- 系统主题和页脚后台配置。
- 审计日志查询页面。
- 自动审核和外部安全数据源接入。

## 完成标准

- 管理员可以完成首次初始化。
- 初始化后系统写入固定内置权限模型。
- 管理员可以登录系统。
- 管理员可以通过最小用户创建入口创建普通用户。
- 管理员可以创建短链。
- 普通用户可以登录并创建短链。
- 未授权的 `guest` 默认不能创建短链，并能看到清晰提示。
- 用户可以复制并访问自己创建的短链。
- 系统可以根据访问域名和短码跳转到目标 URL。
- 用户可以查看、禁用和软删除自己的短链。
- 管理员可以查看和管理全站短链。
- 用户不能创建与当前已知系统固定路由冲突的短码。
- 非法协议或明显不合法的目标链接不能被创建为短链。
- 用户可以切换简体中文和英语。
- 用户可以使用跟随系统、浅色模式和深色模式。
