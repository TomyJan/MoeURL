# MoeURL 产品总览

## 产品定位

MoeURL 是一个现代、轻量、可控的自托管短链系统，面向个人、小团队和可控范围内的公开访问场景。

产品标语建议为：

> MoeURL：现代、轻量、可控的自托管短链系统。

系统优先面向自托管和私有化部署场景，强调部署者自有域名、数据控制、权限可控和轻量维护。后续如需扩展公开 SaaS 或商业化能力，应在现有权限、鉴权、审计和风控扩展点之上继续演进。

## 长期核心目标

本节描述 MoeURL 的完整目标形态。当前 v0.0.4 的实际交付范围以 [v0.0.4 范围](./scope-v0.0.4.md) 为准，远期能力不得自动视为当前版本必须实现内容。

MoeURL 的长期核心目标包括：

- 让用户可以创建、管理和统计短链。
- 让管理员可以统一控制用户、用户组、权限、域名、主题和系统行为。
- 让短链访问支持直接跳转、中间页跳转、确认页跳转、访问密码等多种模式。
- 采用 `https://xxx.com/abcd` 这类简洁短链访问形式。
- 让所有可开放能力都通过权限系统控制，而不是散落在各个独立开关中。
- 让系统具备现代化体验，包括多语言、自动深色模式、响应式布局、前端样式主题化和基础品牌自定义。

## 核心设计原则

### 权限优先

MoeURL 的核心控制逻辑是权限系统。

凡是涉及「能不能使用某个功能」「能不能使用某个域名」「能不能使用某种跳转方式」「能不能配置某类跳转页」的问题，都应优先通过权限系统表达。

系统不单独设计「是否允许游客创建短链」这类割裂配置。游客能力由内置的 `guest` 用户组和 `guest` 用户决定。

### 能力即权限

系统中所有重要功能都应抽象为能力，再由用户组权限开放。

例如：

- 创建短链是一种权限。
- 使用某个域名是一种权限。
- 创建直接跳转短链是一种权限。
- 创建中间页短链是一种权限。
- 创建确认页短链是一种权限。
- 使用访问密码是一种权限。
- 使用某个跳转页模板是一种权限。
- 使用某个模板的可编辑元数据是一种权限。
- 查看详细统计是一种权限。
- 创建长期有效短链是一种权限。

为了降低配置复杂度，管理界面可以在细粒度权限之上提供权限包或权限预设。权限包用于降低管理员配置成本，不替代底层权限模型。

### 默认简单，高级折叠

普通用户的主流程应尽可能简单：

```text
粘贴链接 → 生成短链 → 复制使用
```

高级功能放在折叠区域或高级设置中，只在用户有对应权限时可用。

对于短链创建这类常见、普适的核心组件，无权限时不应直接隐藏，而应置灰并给出友好提示，以保持体验一致。

### 现代化体验

默认目标形态具备：

- 多语言。
- 自动深色模式。
- 响应式布局。
- 清晰的空状态、错误状态和加载状态。
- 移动端友好的短链访问页。
- 可配置的页脚、站点名称、Logo 和主题。

其中，多语言和自动深色模式属于前端基础架构，v0.0.1 应实现基础能力；站点品牌、主题和页脚的后台配置可以后续逐步完善。

## 当前优先目标

当前优先目标是 v0.0.4 基础统计前置。

v0.0.1 已完成基础可用闭环。v0.0.2 已完成发布后第一轮质量补强，包括管理员用户管理、短链列表筛选、E2E 覆盖和部署验证。

v0.0.3 聚焦产品体验：重建浅色和深色主题，重设计首页首屏和导航，抽象可复用短链生成组件，建立统一控制台 Shell，并把已有短链和用户管理页面迁移到新的控制台框架。

v0.0.4 聚焦基础统计前置：持久化短链成功跳转事件，并在我的短链和全站短链列表中展示总访问量、今日访问量和最近访问时间。完整统计分析页、访问明细、来源/设备/地区维度、中间页跳转、访问密码、二维码和短链过期时间等能力仍属于后续版本规划。
