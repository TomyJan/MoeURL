# MoeURL 产品规划文档

> 文档版本：v0.1  
> 产品规划范围：v0.0.1 至 v0.1.0  
> 当前优先目标：v0.0.1 基础可用闭环

## 1. 文档目的

本文档用于描述 MoeURL 的初版产品规划，重点说明系统的使用流程、功能模块、权限思路、页面效果和版本范围。

本文档不涉及具体技术选型、数据库结构、接口设计、部署方式和代码实现。

## 2. 产品定位

MoeURL 定位为一个现代化短链管理系统，面向个人、小团队和可控范围内的公开访问场景。

产品标语建议为：

> MoeURL：现代、轻量、可控的自托管短链系统。

系统优先面向自托管和私有化部署场景，强调部署者自有域名、数据控制、权限可控和轻量维护。后续如需扩展公开 SaaS 或商业化能力，应在现有权限、鉴权、审计和风控扩展点之上继续演进。

系统的核心目标是：

- 让用户可以创建、管理和统计短链。
- 让管理员可以统一控制用户、用户组、权限、域名、主题和系统行为。
- 让短链访问支持直接跳转、中间页跳转、确认页跳转、访问密码等多种模式。
- 采用 `https://xxx.com/abcd` 这类简洁短链访问形式。
- 让所有可开放能力都通过权限系统控制，而不是散落在各个独立开关中。
- 让系统具备现代化体验，包括多语言、自动深色模式、响应式布局、前端样式主题化和基础品牌自定义。

## 3. 核心设计原则

### 3.1 权限优先

MoeURL 的核心控制逻辑是权限系统。

凡是涉及「能不能使用某个功能」「能不能使用某个域名」「能不能使用某种跳转方式」「能不能配置某类跳转页」的问题，都应优先通过权限系统表达。

系统不单独设计「是否允许游客创建短链」这类割裂配置。游客能力由内置的 `guest` 用户组和 `guest` 用户决定。

### 3.2 内置 guest 身份

系统内置两个不可删除对象：

- `guest` 用户组
- `guest` 用户

所有未登录访问者都视为 `guest` 用户，并继承 `guest` 用户组的权限。

`guest` 用户组默认启用，但默认不开放任何实际操作能力。也就是说，未登录访问者默认可以浏览首页、登录、切换语言和主题，但不能创建短链、使用域名、查看统计或执行其他需要授权的操作。

后续如需开放游客能力，应直接调整 `guest` 用户组权限，例如：

- 是否允许创建短链。
- 是否允许自定义短码。
- 是否允许使用指定域名。
- 是否允许使用指定跳转方式。
- 是否允许创建带访问密码的短链。
- 是否允许查看短链统计。
- 每日可创建数量。
- 短链默认有效期。

这样，游客能力不是特殊逻辑，而是权限系统中的一种普通授权结果。

`guest` 用户是系统内部身份，不允许直接登录，不允许修改资料，也不允许删除。若后续开放游客创建短链能力，游客短链默认归属于内置 `guest` 用户；是否支持游客短链认领到登录用户，属于后续扩展能力，初版不要求实现。

### 3.3 短链访问形式

MoeURL 使用简洁的路径型短链访问形式，例如：

```text
https://xxx.com/abcd
```

其中 `xxx.com` 是系统中已配置并启用的短链访问域名，`abcd` 是短码。访问者打开该地址后，系统根据访问域名和短码识别对应短链，并按短链配置执行跳转、展示中间页、展示确认页或要求输入访问密码。

短码在全系统范围内全局唯一，不随短链访问域名变化而重复。也就是说，同一个短码不能在不同短链访问域名下重复使用。

系统页面路径和短链短码路径需要有明确优先级：系统固定路由优先，短链短码路由其次，最后才进入 404 状态页。系统应在短码创建流程中预留保留路径校验入口。v0.0.1 可以先覆盖当前已知固定路由，完整保留路径策略后续继续完善。

### 3.4 本地用户系统与外部登录扩展

系统保留本地用户系统，用于支持基础账号、用户资料、用户组关系和权限计算。

v0.0.1 优先实现本地登录，并预留外部登录扩展入口。在没有可用外部登录方式前，不应提供禁用本地登录的配置，避免管理员误操作导致系统无法登录。

后续登录方式可以扩展为：

- 一个或多个 OIDC 提供商登录。
- 其他后续可扩展的外部认证方式。

OIDC 不是一个简单的「启用 / 禁用 OIDC 登录」开关，而是一组可管理的身份提供商配置。管理员可以添加多个 OIDC 提供商，并分别配置启用状态、显示名称、图标、客户端信息、回调地址、作用域、用户信息映射和默认用户组等。

配置 OIDC 提供商时，系统可以提供预置模板，帮助管理员快速填写常见平台的标准参数。管理员选择模板后，系统自动带出默认授权地址、Token 地址、用户信息地址、默认 Scope 和字段映射，再由管理员补充 Client ID、Client Secret 等实例级信息。

即使后续禁用本地登录，系统内部仍保留本地用户资料，用于绑定外部身份、记录短链归属、计算用户组权限和保存个人偏好。

### 3.5 能力即权限

系统中所有重要功能都应抽象为能力，再由用户组权限开放。

例如：

- 创建短链是一种权限。
- 使用某个域名是一种权限。
- 创建直接跳转短链是一种权限。
- 创建中间页短链是一种权限。
- 创建确认页短链是一种权限。
- 使用访问密码是一种权限。
- 使用某个跳转页模板是一种权限。
- 使用某个模板的可编辑元数据是一种权限。
- 查看详细统计是一种权限。
- 创建长期有效短链是一种权限。

这可以让系统保持一致，也方便后续扩展用户组、套餐、团队和高级权限。

为了避免权限项过多导致配置复杂，管理界面可以在细粒度权限之上提供权限包或权限预设。权限包用于降低管理员配置成本，不替代底层权限模型。例如：

- 基础短链能力：创建短链、使用随机短码、使用默认域名、使用直接跳转。
- 高级短链能力：自定义短码、设置有效期、使用访问密码、使用中间页。
- 统计能力：查看基础统计、查看维度统计、导出统计数据。
- 管理能力：管理用户、管理用户组、管理域名、管理全部短链、管理系统设置。

管理员可以先套用权限包，再按需调整具体权限项。

### 3.6 默认简单，高级折叠

普通用户的主流程应尽可能简单：

```text
粘贴链接 → 生成短链 → 复制使用
```

高级功能放在折叠区域或高级设置中，只在用户有对应权限时可用。

对于短链创建这类常见、普适的核心组件，无权限时不应直接隐藏，而应置灰并给出友好提示，以保持体验一致。例如：

- 「当前用户组暂不可创建短链，请登录或联系管理员开通权限。」
- 「该域名不可用：当前用户组未被授权使用。」
- 「该跳转方式不可用：当前用户组未开放此能力。」

### 3.7 现代化体验

默认目标形态具备：

- 多语言。
- 自动深色模式。
- 响应式布局。
- 清晰的空状态、错误状态和加载状态。
- 移动端友好的短链访问页。
- 可配置的页脚、站点名称、Logo 和主题。

其中，多语言和自动深色模式属于前端基础架构，v0.0.1 应实现基础能力；站点品牌、主题和页脚的后台配置可以后续逐步完善。

## 4. 用户角色与权限体系

### 4.1 用户类型

系统初版建议包含以下用户类型：

| 类型     | 说明                                       |
| -------- | ------------------------------------------ |
| guest    | 内置游客用户，代表所有未登录访问者。       |
| 普通用户 | 通过本地登录或外部登录进入系统的常规用户。 |
| 管理员   | 拥有系统管理能力的用户。                   |

后续可以扩展更多类型，例如团队成员、审计员、高级用户等。

### 4.2 用户组

用户组用于统一配置权限。

系统至少包含：

| 用户组 | 说明                                                                   |
| ------ | ---------------------------------------------------------------------- |
| guest  | 内置不可删除用户组，控制未登录用户能力；默认启用但默认无实际操作权限。 |
| user   | 普通登录用户默认用户组；默认具备创建短链的基础能力。                   |
| admin  | 管理员用户组；具备系统最高管理能力。                                   |

v0.0.1 阶段，一个用户只启用一个主用户组。用户最终权限由主用户组决定，初版不设计针对单个用户的功能授权例外。

工程模型不应把「单用户组」写死为不可扩展约束。后续如需支持团队空间、组织角色、审计员、高级用户或套餐能力，可以在主用户组模型之上演进为多用户组、团队角色或组织级权限。

普通用户组默认开放最基础的短链生成能力。可用域名、可用跳转方式、访问密码、统计级别、模板等能力，均通过用户组权限进一步配置。

为了降低管理员的配置心智负担，系统可以提供预置权限模型，用于快速套用常见部署场景。例如：

- 公开展示站点：`guest` 默认不可创建，仅展示首页和登录入口。
- 公开免费站点：`guest` 可创建基础短链，但受数量、频率、域名和跳转方式限制。
- 仅登录用户：`guest` 不可创建，普通用户可创建和管理自己的短链。
- 团队内部：仅指定用户组可创建短链，并限制可用域名和跳转方式。
- 管理员私有：仅 `admin` 用户组可创建和管理短链。

预置权限模型只是快速初始化和批量配置工具，不替代底层精细权限控制。管理员套用预设后，仍应能继续调整每一个具体权限项。

### 4.3 权限分类

权限可以按模块划分。

#### 短链创建权限

- 创建短链。
- 自定义短码。
- 使用随机短码。
- 设置短链标题。
- 设置短链备注。
- 设置短链标签。
- 设置短链有效期。
- 设置最大访问次数。
- 创建长期有效短链。

有效期相关能力边界如下：

- 设置短链有效期：允许用户手动选择过期时间。
- 创建长期有效短链：允许用户选择无固定过期时间。
- 短链默认有效期：用户未手动设置时，由系统或用户组提供的默认值。

#### 跳转方式权限

跳转方式权限用于控制用户在创建短链时可以选择哪些访问模式：

- 使用直接跳转。
- 使用中间页跳转。
- 使用确认页跳转。
- 使用访问密码。
- 使用指定跳转页模板。

普通用户只能选择自己可用的跳转方式和模板，不能任意自定义系统跳转页。

系统中有权限的用户组可以配置跳转页模板、样式、图片、默认文案、倒计时、按钮样式等。此类配置属于系统模板管理能力，通常只开放给管理员或受信任的高级用户组。

同时，可以保留「基于系统模板，在生成指定链接时进行有限样式定制」的能力。建议按模板定义可编辑元数据，并由权限控制哪些用户组可以填写：

- 模板由系统统一维护，保证基础结构、品牌和安全边界一致。
- 模板声明可编辑字段，例如背景图、提示文案、按钮文案、倒计时、展示图片等。
- 用户创建短链时选择某个模板后，页面显示该模板对应的元数据表单。
- 普通用户只能填写被模板允许、且自己有权限使用的字段。
- 管理员可以配置模板默认值、字段是否开放、字段校验规则和适用用户组。

这样既能避免普通用户随意修改跳转页造成体验不一致，也能保留按链接微调样式的灵活性。

#### 域名权限

- 使用指定域名。
- 设置个人默认域名。
- 管理域名。

域名使用权限严格基于用户组分配。不存在「管理员专属域名」这类独立逻辑；如果某个域名只允许管理员使用，应将它配置为仅 `admin` 用户组可用。

初版不允许用户绑定自己的域名。所有可用域名由管理员统一管理，并按用户组分配使用范围。

#### 统计权限

- 查看自己的短链基础统计。
- 查看自己的短链基础维度统计。
- 查看自己的短链完整详细统计。
- 导出统计数据。
- 查看全站统计。

#### 模板与主题权限

- 使用系统跳转页模板。
- 使用指定模板的可编辑元数据。
- 管理跳转页模板。
- 管理站点主题。
- 管理系统页脚。
- 设置自定义 Open Graph 信息。

系统页脚配置属于主题配置的一部分，只允许管理员配置。普通用户不应拥有独立修改系统页脚的能力。

#### 管理权限

- 管理用户。
- 管理用户组。
- 配置用户组权限。
- 管理全部短链。
- 管理全部域名。
- 管理系统配置。
- 管理系统主题。
- 管理登录、安全和访问策略。
- 查看审计日志和操作记录。

## 5. 整体使用流程

### 5.1 未登录访问首页

未登录用户进入首页时，系统以 `guest` 身份判断可见内容和可用功能。

首页的短链创建区域作为核心组件应保持展示。若 `guest` 拥有创建短链权限，则表单可直接使用；若 `guest` 没有创建短链权限，则表单置灰，并展示清晰提示和登录入口。

首页具体布局见「6.1 首页」。本节只说明未登录状态下的权限判断和交互原则。

### 5.2 登录流程

用户点击登录后，根据系统配置进入本地登录或外部认证流程。

登录成功后，系统创建或更新本地用户资料，并根据用户所属用户组计算权限。

登录后默认进入首页，因为最常用的生成链接功能就在首页。首页应根据登录状态和权限展示可用功能、禁用项提示和用户入口。

用户可以从首页进入用户中心、我的短链、统计分析和个人设置等页面。

### 5.3 创建短链流程

创建短链功能默认放在首页。用户也可以从其他页面通过快捷入口回到首页进行创建。

系统根据权限动态控制可用字段。

基础流程：

```text
输入目标链接
  ↓
选择可用域名
  ↓
选择短码生成方式
  ↓
选择跳转方式
  ↓
选择跳转页模板（如适用）
  ↓
填写模板元数据（如适用）
  ↓
配置可用高级选项
  ↓
生成短链
  ↓
复制或继续管理
```

页面需要做到：

- 对常见核心字段，无权限时优先置灰并说明原因。
- 对高阶或不相关字段，可根据权限隐藏。
- 常用字段默认展开。
- 高级设置默认折叠。
- 创建成功后提供复制、打开、继续创建和查看详情入口。

目标链接在创建和编辑时必须通过基础安全校验。v0.0.1 至少应要求目标链接是合法 URL，并默认只允许 `http://` 和 `https://` 协议；禁止 `javascript:`、`data:`、`file:` 等不适合作为跳转目标的协议。系统还应预留目标链接审核状态和拦截点，便于后续加入黑名单、白名单、外部安全数据源、人工复审和风险链接强制确认页。

v0.0.1 默认不开放 `guest` 创建短链。若后续开放公开创建能力，需要增加更严格的目标地址风险校验，包括内网地址、localhost、私有 IP 段、当前系统管理路径、控制字符、不可见字符和 URL 最大长度等边界。

系统应避免明显的短链循环，例如短链直接指向自身。是否允许短链指向其他短链，可以由后续风控策略进一步限制。

### 5.4 访问短链流程

访问者打开短链后，系统按以下顺序处理：

```text
识别域名和短码
  ↓
查找短链
  ↓
记录 short_link_opened 事件
  ↓
检查短链状态
  ↓
检查访问条件
  ↓
根据检查结果记录 redirect_blocked 或继续处理
  ↓
按跳转方式展示或跳转
  ↓
记录 redirect_initiated / redirect_response_sent 等事件
```

可能结果包括：

- 直接跳转到目标链接。
- 展示中间页后自动跳转。
- 展示确认页，等待用户点击后跳转。
- 要求输入访问密码。
- 显示短链不存在页面。
- 显示短链已禁用页面。
- 显示短链已过期页面。
- 显示访问次数已达上限页面。

所有访问页都应支持多语言、自动深色模式、响应式布局，并使用系统主题和页脚配置。

## 6. 页面规划

### 6.1 首页

首页用于向访问者介绍 MoeURL，并提供登录和快速创建入口。

首页内容根据用户权限动态变化。短链创建区域保持展示，有权限时可用，无权限时置灰并提示。

首页建议采用以下结构：

- 顶部菜单：左侧展示站点 Logo 和站点名称，右侧展示登录入口或头像昵称、语言切换和主题切换。
- 页面主体首屏：展示大标题、简短标语和快速创建短链区域。
- 创建区域：默认只展示最核心的目标链接输入和生成按钮，高级选项默认折叠。
- 权限提示：无权限时保留表单结构，但将不可用字段或按钮置灰，并给出登录或联系管理员提示。
- 下翻内容：展示产品介绍、功能说明、使用场景和页脚信息。

建议包含：

- 站点 Logo。
- 站点名称。
- 简短标语。
- 登录按钮或当前用户入口。
- 快速创建短链区域。
- 权限限制提示。
- 功能介绍。
- 语言切换。
- 主题切换。
- 页脚区域。

首页应适配桌面端和移动端。

### 6.2 登录页与登录结果页

登录页根据系统配置展示可用登录方式。

v0.0.1 阶段仅要求本地账号登录；外部登录入口在对应登录方式实现后展示。

完整目标形态可以包含：

- 本地账号登录。
- 已启用的 OIDC 提供商登录入口。
- 其他已启用的认证入口。

后续当至少存在一个可用外部登录方式时，管理员可以选择禁用本地登录。禁用后，登录页隐藏本地账号表单，只展示已启用的外部登录入口。

当配置了多个 OIDC 提供商时，登录页应展示清晰的提供商列表，例如「使用 GitHub 登录」「使用 Google 登录」「使用企业 SSO 登录」。

用户完成登录后，系统可以进入登录结果页，也可以直接跳转首页。

常见状态：

- 登录成功。
- 登录失败。
- 用户被禁用。
- 当前账号无权使用系统。
- 首次登录，需要补充昵称或偏好设置。

其中「用户被禁用」表示本地账号状态不可用；「当前账号无权使用系统」主要用于外部登录场景，表示外部认证成功，但该账号没有匹配允许进入系统的用户组或访问规则。

### 6.3 用户中心首页

### 6.4 我的短链页

- 跳转方式。
- 访问次数。

操作包括：

- 复制。
- 打开。
- 编辑。
- 查看统计。
- 启用或禁用。
- 删除。

筛选条件：

- 关键词。

### 6.5 短链详情页

## 7. 短链跳转模式

本节描述 MoeURL 跳转模式的完整目标形态，具体实现顺序以「13. 初版范围建议」为准。

### 7.1 直接跳转

访问短链后直接进入目标链接。

适合：

- 工具类短链。
- 内部分享。
- 对速度要求高的场景。

页面效果：

- 用户通常看不到中间页面。
- 系统仍可记录访问事件。

### 7.2 中间页跳转

访问短链后先展示中间页，再自动跳转。

中间页可包含：

- 目标域名。
- 倒计时。
- 立即跳转按钮。
- 安全提示。
- 背景图。
- 自定义文案。
- 页脚。

适合：

- 外链提醒。
- 广告展示。
- 品牌曝光。
- 需要缓冲提示的场景。

### 7.3 确认页跳转

访问短链后需要用户主动确认。

确认页可包含：

- 目标链接或目标域名。
- 风险提示。
- 继续访问按钮。
- 返回按钮。

适合：

- 外部未知链接。
- 风险链接提示。
- 需要访问者主动确认后再继续访问的链接。
- 管理员要求手动确认的链接。

### 7.4 访问密码

访问短链后要求输入密码，验证通过后再根据短链配置进入直接跳转、中间页或确认页。

访问密码是否可用由用户组权限控制。

访问密码只保存哈希值，不保存明文。管理员和创建者均不能查看原密码，只能重置密码。密码验证失败不应计入有效访问量，但可以记录为访问条件失败事件，用于后续风控、审计或统计分析。

## 8. 多域名管理

### 8.1 域名作为资源

域名是系统中的可管理资源。

域名可以按用途分为：

- 系统访问域名：用于访问首页、登录页、用户中心和管理后台。
- 短链访问域名：用于承载 `https://xxx.com/abcd` 形式的短链访问。

v0.0.1 可以允许系统访问域名和短链访问域名共用，但模型上应保留用途区分，便于后续支持独立管理后台域名、独立短链域名或多个短链访问域名。

v0.0.1 阶段只要求支持一个默认短链访问域名。域名可以先作为系统配置存在，不要求完整域名管理页面。但短链记录应保存域名信息或域名引用，避免后续扩展多域名时重构短链主表。

每个域名可以配置：

- 域名名称。
- 显示名称。
- 是否启用。
- 默认跳转方式。
- 默认跳转页模板。
- 可用用户组。
- 说明文案。
- 404 页面策略。
- 排序值。

### 8.2 域名使用权限

用户创建短链时，只能选择自己有权限使用的域名。

域名开放范围严格基于用户组配置：

- 可配置为所有用户组可用。
- 可配置为指定用户组可用。
- 可配置为仅 `admin` 等特定用户组可用。

不设计指定用户级别的域名授权，也不设计「仅管理员」这类额外开关。类似需求都通过用户组表达。

`guest` 是否可以使用某个域名，也通过域名可用用户组和 `guest` 用户组权限共同决定。

### 8.3 默认域名

系统支持按优先级决定创建短链时的默认域名。

缺省顺序如下：

```text
用户自定义默认域名
  ↓
用户组默认域名
  ↓
全局默认域名
  ↓
域名排序兜底
```

说明：

- 用户自定义默认域名由用户在个人设置中选择，优先级最高。
- 用户组默认域名由管理员在用户组中配置，用于给不同用户组提供合理默认值。
- 全局默认域名由管理员配置，用于系统级兜底。
- 域名排序用于在没有更高优先级默认值，或更高优先级配置不可用时决定候选顺序。
- 所有默认域名都必须满足当前用户的域名使用权限。
- 如果更高优先级配置不可用，应自动回退到下一个可用域名，并在需要时向用户说明回退原因。

## 10. 外观、主题与跳转页模板

### 10.1 主题模式

系统支持 3 种主题模式：

- 跟随系统。
- 浅色模式。
- 深色模式。

未登录访问者默认跟随系统。

登录用户可以在个人设置中保存偏好。

### 10.2 响应式布局

所有核心页面都应适配：

- 手机。
- 平板。
- 桌面。

短链访问页尤其需要优先保证移动端体验。

### 10.3 站点品牌

管理员可以配置：

- 站点名称。
- Logo。
- Favicon。
- 主题色。
- 首页标语。
- 默认背景图。

### 10.4 页脚配置

页脚是系统主题配置的一部分，只允许管理员配置。

可配置内容包括：

- 版权信息。
- 备案号。
- 自定义链接。
- Powered by 文案（可配置是否展示）。
- 简单自定义文本。

页脚配置应用于首页、系统页面、跳转页和错误页。具体页面是否展示某些页脚内容，可以由主题策略控制。

### 10.5 跳转页模板

系统可以内置多种跳转页模板：

- 默认中间页。
- 极简确认页。
- 图片背景页。
- 错误提示页。

用户能否使用某个模板，由用户组权限控制。

每个模板可以声明自己的元数据字段。用户选择某种模板后，创建短链表单会出现该模板对应的填写项。

示例：

| 模板       | 可填写元数据                           |
| ---------- | -------------------------------------- |
| 默认中间页 | 提示文案、倒计时、按钮文案。           |
| 图片背景页 | 背景图、主标题、副标题、按钮文案。     |
| 极简确认页 | 风险提示、确认按钮文案、返回按钮文案。 |

模板元数据设计原则：

- 由模板定义字段，不允许用户随意注入任意页面结构。
- 字段应有类型、默认值、校验规则和权限控制。
- 管理员可以配置模板默认值和适用用户组。
- 普通用户只能在创建或编辑指定短链时填写自己可用的字段。

## 11. 统计、记录与状态页面

统计能力分为 3 层：

- 基础统计：访问总量、今日访问量、趋势、最近访问时间等核心指标。
- 基础维度统计：来源、设备、地区等低成本聚合维度。
- 完整详细统计：最近访问明细、事件明细、复杂筛选、导出等高级能力。

### 11.1 基础统计

基础统计用于满足日常管理需求。

包括：

- 总访问量。
- 今日访问量。
- 近 7 天访问趋势。
- 近 30 天访问趋势。
- 最近访问时间。

### 11.2 维度统计与详细统计

维度统计和详细统计用于分析访问来源。

v0.1.0 可以先实现来源、设备、地区等基础维度统计。完整详细统计后续再实现浏览器、操作系统、最近访问记录、事件明细、导出等能力。

包括：

- 来源页面。
- 浏览器。
- 操作系统。
- 设备类型。
- 地区。
- 最近访问记录。

详细统计展示由权限控制。

### 11.3 统计事件口径

短链访问流程应在工程设计上提前定义统计事件类型，避免后续新增密码页、确认页、中间页、风控审核时造成统计口径混乱。

建议至少预留以下事件：

| 事件                       | 触发时机                                                     |
| -------------------------- | ------------------------------------------------------------ |
| `short_link_opened`        | 访问者打开短链 URL，并成功识别域名和短码。                   |
| `access_condition_checked` | 系统完成访问条件检查，例如过期、禁用、次数上限、密码要求等。 |
| `password_verified`        | 访问者输入访问密码且验证通过。                               |
| `confirmation_clicked`     | 访问者在确认页点击继续访问。                                 |
| `redirect_initiated`       | 系统准备跳转到目标链接。                                     |
| `redirect_response_sent`   | 系统已返回跳转响应，或跳转页已触发跳转动作。                 |
| `redirect_blocked`         | 短链因状态、权限、风控或访问条件被阻止。                     |

`redirect_response_sent` 只表示 MoeURL 已完成自身可确认的跳转处理，不代表访问者浏览器一定完成跳转，也不代表目标页面实际加载成功。

后台处理流程应允许不同事件分别入库或聚合。基础统计可以只展示访问量和趋势，详细统计再展示来源、设备、地区、事件类型和最近访问记录。

### 11.4 操作记录

系统应记录关键操作，便于追溯。

建议记录：

- 创建短链。
- 编辑短链。
- 删除短链。
- 禁用短链。
- 修改权限。
- 修改用户组。
- 修改域名配置。
- 修改系统设置。

### 11.5 短链状态模型

系统应为短链定义统一状态，避免访问流程、管理后台和统计口径各自判断。

建议状态包括：

| 状态        | 说明                                     |
| ----------- | ---------------------------------------- |
| `active`    | 启用中，可以按配置访问。                 |
| `disabled`  | 已禁用，不允许访问目标链接。             |
| `expired`   | 已过期，不允许访问目标链接。             |
| `exhausted` | 访问次数已达上限，不允许访问目标链接。   |
| `pending`   | 待审核，预留给后续风控和人工审核。       |
| `rejected`  | 审核拒绝，预留给后续风控和人工审核。     |
| `deleted`   | 已删除或软删除，不作为普通列表默认展示。 |

访问短链时，应先识别短链，再根据状态决定继续检查访问条件、展示状态页或返回 404。删除策略采用软删除，便于审计，并防止短码被复用造成误导。

v0.0.1 删除短链采用软删除。删除后普通列表默认不展示，访问时按不存在处理，短码永久保留占用，不进入自动复用池。v0.0.1 不提供短码释放、恢复或复用能力。

### 11.6 状态页面

短链访问过程中可能出现多种状态。系统应提供统一、可读、可自定义的状态页。

常见状态页：

- 短链不存在。
- 短链已禁用。
- 短链已过期。
- 访问次数已达上限。
- 需要访问密码。
- 密码错误。
- 无权访问。
- 目标链接不可用。
- 系统维护中。

状态页面应支持：

- 多语言。
- 自动深色模式。
- 响应式布局。
- 系统主题和页脚配置。
- 返回首页按钮。

### 11.7 风控与审核扩展入口

风控和滥用治理属于远期开发目标，v0.0.1 不要求完整实现，但工程设计应预留扩展入口。

本节风控主要针对目标链接和短链滥用治理，不替代登录安全、账号安全和系统访问策略。

后续可扩展能力包括：

- 目标域名黑名单和白名单。
- 自动审核规则。
- 接入 Google 安全浏览等外部安全数据源进行目标链接审核。
- 管理员手动复审。
- 用户举报入口。
- 可疑短链标记。
- 创建频率限制。
- 每日创建数量限制。
- 风险链接强制确认页。
- 审核状态和审核记录。

短链创建、编辑和访问流程中应预留审核状态字段和拦截点，便于后续在不重构主流程的情况下加入自动审核、白名单、外部安全数据源和人工复审。

## 12. 管理后台规划

### 12.1 系统仪表盘

系统仪表盘用于展示全站概况。

建议包含：

- 用户总数。
- 短链总数。
- 今日新增短链。
- 今日访问量。
- 近期访问趋势。
- 系统状态摘要。

### 12.2 域名与链接

链接管理用于管理全站短链。

管理员可以：

- 查看全部短链。
- 搜索全部短链。
- 按用户、域名、状态筛选。
- 禁用短链。
- 删除短链。
- 修改短链所属用户。
- 查看短链统计。

域名管理用于管理系统可用域名。

管理员可以：

- 添加域名。
- 编辑域名。
- 禁用域名。
- 删除域名。
- 设置全局默认域名。
- 配置域名可用用户组。
- 配置域名默认跳转方式和默认跳转页模板。
- 配置域名排序。

### 12.3 用户与权限

用户管理用于维护本地用户资料和账号状态。

管理员可以：

- 查看用户列表。
- 搜索用户。
- 查看用户详情。
- 启用或禁用用户。
- 调整用户组。
- 查看用户短链数量。
- 查看用户访问量。

用户组管理用于配置权限。

管理员可以：

- 查看用户组。
- 创建用户组。
- 编辑用户组。
- 配置用户组权限。
- 配置用户组可用域名。
- 配置用户组可用跳转方式和模板。
- 配置用户组默认域名。
- 查看用户组成员。

内置用户组不可删除，但可以修改部分权限。

用户组编辑页建议按能力分组展示，避免权限项平铺导致难以理解：

```text
用户组基础信息
  名称
  标识
  描述
  是否内置

短链能力
  创建短链
  自定义短码
  设置标题、备注、标签
  设置有效期
  使用访问密码

域名权限
  可用域名
  默认域名

跳转能力
  直接跳转
  中间页跳转
  确认页跳转
  可用跳转页模板
  可编辑模板元数据

统计权限
  基础统计
  维度统计
  详细统计
  导出统计

管理权限
  用户管理
  用户组管理
  全站短链管理
  域名管理
  系统设置
  主题设置
  审计日志
```

权限包和权限预设可以作为用户组编辑页中的快捷配置入口，但最终仍应落到具体权限项。

### 12.4 系统维护

系统维护用于管理全局配置、主题和审计记录。

本节描述系统维护的完整目标形态，具体实现顺序以第 13 节为准。

系统设置建议包含：

- 默认语言。
- 默认主题。
- 默认短码长度。
- 默认跳转方式。
- 默认中间页倒计时。
- 默认短链有效期。
- 默认用户组。
- 本地登录开关（需存在可用外部登录方式时才允许关闭）。
- 外部接口鉴权扩展入口，用于后续支持程序化创建短链、查询短链和获取统计数据。
- OIDC 提供商管理。
- OIDC 提供商预置模板。

主题设置建议包含：

- 站点名称。
- Logo。
- Favicon。
- 主题色。
- 首页文案。
- 页脚内容。
- 默认跳转页模板。

审计日志 / 操作记录用于查询关键操作历史。

注意：具体功能是否可用，仍应由权限系统最终决定。

### 12.5 首次初始化流程

自托管部署首次启动时，如果系统不存在管理员账号，应进入初始化流程。初始化完成前，普通登录、短链创建和管理后台应不可用。

初始化内容至少包括：

- 注册第一个用户，该用户加入内置 `admin` 用户组。
- 设置站点名称。
- 配置默认系统访问域名或确认当前访问域名。
- 配置默认短链访问域名。
- 选择默认语言。
- 选择基础权限模型。

初始化完成后，第一个用户默认属于 `admin` 用户组，并拥有系统最高管理权限。系统应禁止重复执行初始化流程，除非管理员通过明确的维护机制重置系统状态。

## 13. 初版范围建议

版本规划按「基础可用闭环 → MVP 增强闭环 → 增强能力」逐步推进。v0.0.1 不是临时 Demo，而是具备长期演进基础的第一版可用系统。即使某些能力放到后续版本，v0.0.1 的工程设计也应预留必要抽象：权限、鉴权、统计事件、审核状态、模板元数据、后台处理流程都应留好扩展入口，避免后续新增能力时重构主流程。

这里的「后续实现」不代表初版可以写成临时逻辑。初版实现时仍应按长期架构预留必要抽象，只是暂不开放完整功能或完整管理界面。

### 13.1 v0.0.1 基础可用闭环

v0.0.1 的目标是让系统完成从初始化、本地登录、创建短链到访问短链的基础可用闭环，并建立后续演进所需的基础架构。

v0.0.1 不追求完整后台和完整权限管理界面。凡是会显著拉高第一版复杂度、但不影响基础功能闭环的能力，都应优先作为模型预留或后续版本实现。

#### 必须实现

- 首次初始化流程。
- 本地登录流程。
- 本地用户系统的最小模型。
- 内置 `guest` 用户和 `guest` 用户组。
- 内置 `user` 用户组和 `admin` 用户组。
- v0.0.1 阶段一个用户只启用一个主用户组。
- 最小权限判断。
- 首页。
- 首页创建短链。
- 目标链接基础安全校验。
- 直接跳转。
- 我的短链列表。
- 复制短链、打开短链、启用或禁用短链、软删除短链。
- 管理员查看和管理全站短链的最小入口。
- 单默认短链访问域名。
- 系统固定路由与短链短码路由的优先级处理。
- 保留路径校验入口，v0.0.1 先覆盖当前已知固定路由。
- 响应式基础布局。
- 多语言基础架构，至少支持简体中文和英语。
- 主题模式基础架构，至少支持跟随系统、浅色模式和深色模式。

#### v0.0.1 最小权限集合

v0.0.1 只要求实现支撑基础闭环的权限项。其他权限可以先在模型设计上预留，不要求提供完整可视化配置。

建议最小权限项包括：

- `short_link:create`：创建短链。
- `short_link:read_own`：查看自己的短链。
- `short_link:update_own`：编辑、启用或禁用自己的短链。
- `short_link:delete_own`：删除自己的短链。
- `short_link:manage_all`：管理全站短链。
- `domain:use_default`：使用默认短链访问域名。
- `admin:access`：访问基础管理入口。

`guest` 用户组默认不授予上述实际操作权限。`user` 用户组默认可拥有自己的短链创建和管理权限。`admin` 用户组默认拥有系统最高管理权限。

#### 应该预留

- 登录方式扩展入口。
- OIDC 提供商模型或配置扩展点。
- 用户组权限管理的扩展结构。
- 多域名模型和域名权限扩展结构。
- 统计事件类型、事件触发点和后台处理流程边界；v0.0.1 不要求实现统计展示、统计聚合或访问明细查询。
- 短链状态模型。
- 审核状态字段和风控拦截点。
- 跳转页模板和模板元数据结构。
- 外部接口鉴权扩展入口。
- 操作记录扩展点。

#### 暂不实现

- OIDC 登录。
- 禁用本地登录。
- 中间页跳转。
- 确认页跳转。
- 访问密码。
- 二维码能力。
- 完整用户中心仪表盘。
- 短链详情完整页。
- 统计页、统计聚合和访问明细查询。
- 完整域名管理页面。
- 用户组权限的完整可视化配置。
- 权限预设模型的完整可视化配置。
- 系统主题和页脚后台配置。
- 审计日志查询页面。
- 自动审核和外部安全数据源接入。

#### 完成标准

- 管理员可以完成首次初始化。
- 管理员可以登录系统。
- 管理员可以创建短链。
- 普通用户可以登录并创建短链。
- 未授权的 `guest` 默认不能创建短链，并能看到清晰提示。
- 用户可以复制并访问自己创建的短链。
- 系统可以根据访问域名和短码跳转到目标 URL。
- 用户可以查看、禁用和软删除自己的短链。
- 管理员可以查看和管理全站短链。
- 用户不能创建与当前已知系统固定路由冲突的短码，后续保留路径策略可在既有校验入口上继续完善。
- 非法协议或明显不合法的目标链接不能被创建为短链。
- 用户可以切换简体中文和英语。
- 用户可以使用跟随系统、浅色模式和深色模式。
