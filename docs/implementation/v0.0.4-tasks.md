# v0.0.4 任务级实施清单

> 本文档把 [v0.0.4 实施计划](./v0.0.4-plan.md) 拆成可执行任务。实现时必须遵循技术基线、TDD 和覆盖率门禁。

## 1. 当前状态

v0.0.3 已完成 UI/UX 精细化重设计。当前 v0.0.4 聚焦基础统计前置，不新增完整统计分析页。

v0.0.4 的优先级顺序是：

1. 文档和范围同步。
2. 数据库事件表和 SQLC 查询。
3. 访问事件记录和后端统计响应。
4. 前端模型和列表展示。
5. 测试、E2E 和验收收尾。

## 3. 阶段 2：数据库事件表和 SQLC 查询

### 2.1 新增事件表 migration

建议文件：

- 创建：`migrations/00002_short_link_event.sql`
- 测试：`internal/db/migration_test.go`

验收：

- `short_link_event` 表通过 migration 创建。
- 表使用 `uuid` 主键。
- `short_link_id` 引用 `short_link(id)`。
- `event_type` 保存稳定事件名。
- `created_at` 使用 `timestamptz`。
- 回滚时删除事件表。

### 2.2 新增事件 SQLC 查询

建议文件：

- 创建：`queries/event.sql`
- 修改：`queries/short_link.sql`
- 生成：`internal/db/sqlc/*.go`
- 测试：`internal/db/sqlc_test.go`

验收：

- 可以插入短链事件。
- 我的短链列表查询返回总访问量、今日访问量和最近访问时间。
- 管理员全站短链列表查询返回同样统计。
- 无事件时统计字段稳定为 `0` 和空时间。

## 4. 阶段 3：访问事件记录和后端统计响应

### 3.1 实现数据库事件记录器

建议文件：

- 修改：`internal/event/event.go`
- 新增：`internal/event/recorder.go`
- 新增测试：`internal/event/recorder_test.go`

验收：

- `event.NewRecorder(pool)` 返回写入数据库的 recorder。
- `Record` 为每个事件生成 UUID。
- 只持久化带有短链 ID 的成功事件。
- recorder 写入失败时返回错误，但调用方可以选择忽略。

### 3.2 让跳转流程记录成功访问

建议文件：

- 修改：`internal/shortlink/redirect_service.go`
- 测试：`internal/shortlink/redirect_test.go`

验收：

- 成功访问短链写入 `redirect_response_sent` 事件。
- 禁用短链访问不写入成功访问事件。
- 不存在或软删除短链不写入成功访问事件。

### 3.3 扩展短链服务模型和 handler 响应

建议文件：

- 修改：`internal/shortlink/model.go`
- 修改：`internal/shortlink/service.go`
- 测试：`internal/shortlink/service_test.go`
- 测试：`internal/shortlink/handler_test.go`

验收：

- 列表返回 `stats.visitCount`。
- 列表返回 `stats.todayVisitCount`。
- 列表返回 `stats.lastVisitedAt`。
- 管理员列表和我的短链列表字段一致。
- 创建、更新和删除响应不强制包含统计字段。
