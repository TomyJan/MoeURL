# v0.0.1 验收清单

> 本文档用于 v0.0.1 开发完成后的功能验收。验收时应结合实际技术栈运行自动化测试，并进行关键路径手工验证。
>
> 具体验收口径应与 [v0.0.1 工程实施合同](./v0.0.1-implementation-contract.md) 保持一致。

## 1. 验收原则

- 优先验证基础可用闭环是否完整。
- 不验收 v0.0.1 明确暂不实现的能力。
- 每个失败项都应记录复现步骤、期望结果和实际结果。
- 未经验证的功能不应标记为完成。

## 2. 当前验证记录

以下记录来自 `feature/v0.0.1-foundation` 分支当前实现，用于说明已经跑过的自动化验证。它不替代后续逐项手工验收。

| 验证项 | 命令 | 最近结果 |

| 后端测试 | `go test ./...` | 通过 |
| 前端单元测试 | `cd web && pnpm test` | 11 个测试文件、33 个测试通过 |

当前 E2E 已覆盖：

- 初始化系统。
- 使用初始化管理员账号登录。
- 登录后展示当前用户。
- 首页通过 UI 创建短链。
- 创建成功后展示短链地址，并提供复制、打开和继续创建入口。
- 访问有效短链并跳转到目标地址。
- 禁用短链后访问被阻止。
- 我的短链列表展示短链，并提供复制和打开入口。
- 退出登录后回到 `guest` 状态。
- `guest` 首页创建入口置灰并展示登录提示。

## 3. 环境验收

- [ ] 可以安装依赖。
- [ ] 可以启动开发环境。
- [ ] `go test ./...` 通过。
- [ ] `cd web && pnpm test` 通过。
- [ ] 后端测试覆盖率达到 100%。
- [ ] 前端测试覆盖率达到 100%。
- [ ] `cd web && pnpm build` 通过。
- [ ] `cd web && pnpm test:e2e` 通过。
- [ ] `docker compose up --build` 可以启动应用和 PostgreSQL。
- [ ] 访问 `/api/v1/health` 返回 `ok`。
- [ ] 访问未知路径进入 404 页面。
- [ ] API 业务成功和业务失败使用统一数字 `code` 响应体。
- [ ] `guest` 权限不足返回 HTTP `200`，响应体使用业务错误码表达。
- [ ] 除登录态无效、已登录用户基础访问控制失败和基础设施级 `500` 外，业务错误不依赖 HTTP 状态码表达。

## 4. 首次初始化验收

- [ ] 未初始化系统访问首页时进入初始化流程。
- [ ] 初始化前登录页不可正常进入业务系统。
- [ ] 初始化前管理后台不可用。
- [ ] 初始化表单要求填写管理员账号、密码、站点名称、系统访问域名和默认短链访问域名。
- [ ] 初始化时会校验默认短链访问域名。
- [ ] 初始化成功后创建第一个管理员用户。
- [ ] 第一个管理员用户属于 `admin` 用户组。
- [ ] 初始化成功后系统写入固定内置权限模型。
- [ ] 初始化成功后系统标记为已初始化。
- [ ] 初始化成功后无法重复执行初始化流程。

## 5. 本地登录验收

- [ ] 管理员可以使用初始化账号登录。
- [ ] 普通用户可以使用本地账号登录。
- [ ] 错误密码无法登录。
- [ ] 不存在账号无法登录。
- [ ] 禁用用户无法登录。
- [ ] 登录失败时展示明确错误。
- [ ] 登录成功后进入首页或预期页面。
- [ ] 登录后可以识别当前用户。
- [ ] 退出登录后回到未登录状态。
- [ ] 未登录访问者按 `guest` 身份处理。

## 6. 权限验收

- [ ] 系统内置 `guest` 用户组。
- [ ] 系统内置 `user` 用户组。
- [ ] 系统内置 `admin` 用户组。
- [ ] 系统内置不可登录、不可删除的 `guest` 用户。
- [ ] 内置用户组不可删除。
- [ ] `guest` 默认没有实际操作权限。
- [ ] `user` 默认可以创建和管理自己的短链。
- [ ] `admin` 默认可以访问基础管理入口。
- [ ] `guest` 访问首页时创建区域置灰。
- [ ] `guest` 创建短链请求返回 HTTP `200`，响应体 `code` 为 `120001`。
- [ ] 普通用户不能查看或管理他人的短链。
- [ ] 非管理员不能进入管理入口。
- [ ] 无权限操作不只依赖前端隐藏，服务端也会拒绝。

## 7. 首页和创建短链验收

- [ ] 首页可以正常访问。
- [ ] 首页展示站点名称或 Logo 区域。
- [ ] 首页展示快速创建短链区域。
- [ ] 有权限用户可以输入目标 URL 创建短链。
- [ ] 创建短链默认使用系统默认短链访问域名。
- [ ] 创建短链默认生成随机短码。
- [ ] 系统生成的短码只包含小写字母和数字。
- [ ] 创建成功后展示完整短链地址。
- [ ] 创建成功后可以复制短链地址。
- [ ] 创建成功后可以打开短链地址。
- [ ] 创建成功后可以继续创建下一条短链。

## 8. URL 校验验收

- [ ] 合法 `http://` URL 可以创建短链。
- [ ] 合法 `https://` URL 可以创建短链。
- [ ] 空 URL 不能创建短链。
- [ ] 明显不合法 URL 不能创建短链。
- [ ] `javascript:` URL 不能创建短链。
- [ ] `data:` URL 不能创建短链。
- [ ] `file:` URL 不能创建短链。
- [ ] 编辑目标 URL 时仍会执行同样校验。

## 9. 短码和路由验收

- [ ] 短码全系统唯一。
- [ ] 重复短码不会创建成功。
- [ ] 与系统固定路由冲突的短码不会创建成功。
- [ ] 短码访问查找按小写归一化值处理。
- [ ] 系统固定路由优先于短码路由。
- [ ] 访问不存在短码进入 404 或短链不存在页。
- [ ] 软删除短链后原短码不能再次使用。

## 10. 我的短链验收

- [ ] 用户可以进入我的短链页。
- [ ] 我的短链页只展示当前用户自己的短链。
- [ ] 列表展示短链地址。
- [ ] 列表展示目标 URL。
- [ ] 列表展示状态。
- [ ] 列表展示创建时间。
- [ ] 空列表时展示清晰空状态。
- [ ] 用户可以复制自己的短链。
- [ ] 用户可以打开自己的短链。
- [ ] 用户可以禁用自己的短链。
- [ ] 用户可以重新启用自己的短链。
- [ ] 用户可以软删除自己的短链。
- [ ] 软删除后普通列表不再展示该短链。

## 11. 短链访问验收

- [ ] 访问有效短链会跳转到目标 URL。
- [ ] 访问已禁用短链不会跳转到目标 URL。
- [ ] 访问已禁用短链展示明确状态页。
- [ ] 访问已删除短链按不存在处理。
- [ ] 访问不存在短链展示 404 或短链不存在页。
- [ ] 访问流程使用统一事件记录接口。
- [ ] 访问流程调用 `short_link_opened` 事件。
- [ ] 访问流程调用 `access_condition_checked` 事件。
- [ ] 访问流程调用 `redirect_initiated` 事件。
- [ ] 访问流程调用 `redirect_response_sent` 事件。
- [ ] 访问被阻止时调用 `redirect_blocked` 事件。
- [ ] v0.0.1 不要求事件入库、统计聚合或统计 UI。

## 12. 管理员创建普通用户验收

- [ ] 管理员可以进入创建用户入口。
- [ ] 普通用户不能进入创建用户入口。
- [ ] 管理员可以设置用户名、密码、昵称、用户组和启用状态。
- [ ] 用户名重复时创建失败。
- [ ] 创建用户时不会保存明文密码。
- [ ] 被创建的普通用户可以登录。
- [ ] 被创建的普通用户可以创建并管理自己的短链。

## 13. 管理员全站短链验收

- [ ] 管理员可以进入基础管理入口。
- [ ] 普通用户不能进入基础管理入口。
- [ ] 管理员可以查看全站短链列表。
- [ ] 全站短链列表展示短链地址。
- [ ] 全站短链列表展示目标 URL。
- [ ] 全站短链列表展示所属用户。
- [ ] 全站短链列表展示状态。
- [ ] 全站短链列表展示创建时间。
- [ ] 管理员可以禁用任意短链。
- [ ] 管理员可以启用任意短链。
- [ ] 管理员可以软删除任意短链。
- [ ] 管理员操作受权限保护。

## 14. 多语言验收

- [ ] 系统至少支持简体中文。
- [ ] 系统至少支持英语。
- [ ] 用户可以切换语言。
- [ ] 首页核心文案支持语言切换。
- [ ] 登录页核心文案支持语言切换。
- [ ] 我的短链页核心文案支持语言切换。
- [ ] 基础管理入口核心文案支持语言切换。
- [ ] 短链状态页核心文案支持语言切换。
- [ ] 常见错误提示支持语言切换。

## 15. 主题验收

- [ ] 系统支持跟随系统模式。
- [ ] 系统支持浅色模式。
- [ ] 系统支持深色模式。
- [ ] 用户可以切换主题模式。
- [ ] 默认主题采用 Material Design 3 风格。
- [ ] Vuetify 初始化配置中存在 MoeURL 自定义主题。
- [ ] 浅色和深色主题都定义主色、辅助色、背景色和错误色。
- [ ] 首页在浅色和深色模式下可读。
- [ ] 登录页在浅色和深色模式下可读。
- [ ] 我的短链页在浅色和深色模式下可读。
- [ ] 短链状态页在浅色和深色模式下可读。

## 16. 响应式验收

- [ ] 首页在移动端可用。
- [ ] 登录页在移动端可用。
- [ ] 我的短链页在移动端可读、可操作。
- [ ] 短链状态页在移动端可读。
- [ ] 移动端可以完成登录。
- [ ] 移动端可以创建短链。
- [ ] 移动端可以复制短链。
- [ ] 移动端可以查看短链列表。

## 17. PWA 验收

- [ ] 应用提供 Web App Manifest。
- [ ] 应用提供图标和启动主题色。
- [ ] Manifest 包含应用名称、图标、启动 URL、显示模式和主题色。
- [ ] Service Worker 注册成功。
- [ ] 应用可以作为 PWA 安装。
- [ ] 移动端安装后启动显示正常。
- [ ] App Shell 和静态资源可以按策略缓存。
- [ ] `/api/v1/*` 响应、登录态 API、短链业务数据、短链跳转业务响应和权限相关响应不会被缓存。

## 18. 暂不实现项确认

以下能力不应作为 v0.0.1 验收阻塞项：

- [ ] 不要求用户自助注册。
- [ ] 不要求 OIDC 登录。
- [ ] 不要求禁用本地登录。
- [ ] 不要求中间页跳转。
- [ ] 不要求确认页跳转。
- [ ] 不要求访问密码。
- [ ] 不要求二维码能力。
- [ ] 不要求完整用户中心仪表盘。
- [ ] 不要求短链详情完整页。
- [ ] 不要求统计页、统计聚合和访问明细查询。
- [ ] 不要求完整域名管理页面。
- [ ] 不要求用户组权限的完整可视化配置。
- [ ] 不要求权限预设模型的完整可视化配置。
- [ ] 不要求系统主题和页脚后台配置。
- [ ] 不要求审计日志查询页面。
- [ ] 不要求自动审核和外部安全数据源接入。

- [ ] 所有自动化测试通过。
- [ ] 构建通过。
- [ ] 基础手工验收完成。
- [ ] 文档与实际功能一致。
- [ ] README 中的启动命令可用。
- [ ] 没有把暂不实现能力误标为已完成。
