## 2. v0.0.1 范围裁决

### 2.1 普通用户来源

v0.0.1 必须支持普通用户登录和创建短链，因此必须提供普通用户来源。

裁决：v0.0.1 提供管理员创建普通用户的最小入口，不实现完整用户管理系统。

最小入口支持：

- 创建用户。
- 设置用户名、密码、昵称。
- 选择用户组。
- 设置启用状态。

不支持：

- 用户自助注册。
- 批量导入。
- 用户资料完整编辑。
- 用户删除。
- 多用户组。
- OIDC 绑定。

### 2.2 初始化权限模型

v0.0.1 不提供权限模型选择器。初始化时写入固定内置权限模型：

- `guest` 用户组：不能创建短链。
- `user` 用户组：可以创建和管理自己的短链，可以使用默认短链访问域名。
- `admin` 用户组：拥有 v0.0.1 全部权限。

### 2.3 短链详情页

v0.0.1 不实现独立短链详情页。

创建成功后提供：

- 复制短链。
- 打开短链。
- 继续创建。
- 返回我的短链列表。

我的短链列表承担基础查看和操作入口。

### 2.4 事件预留

v0.0.1 预留统一事件记录接口和事件类型，不要求事件入库、聚合、查询 API 或统计 UI。

必须预留事件：

- `short_link_opened`
- `access_condition_checked`
- `redirect_blocked`
- `redirect_initiated`
- `redirect_response_sent`

### 2.5 短链状态

v0.0.1 必须实现：

- `active`
- `disabled`
- 软删除，通过 `deleted_at` 表达

后续预留：

- `expired`
- `exhausted`
- `pending`
- `rejected`

### 2.6 域名策略

v0.0.1 仅使用全局默认短链访问域名。

不实现：

- 用户默认域名。
- 用户组默认域名。
- 完整域名管理页面。
- 域名排序兜底策略。

## 3. 页面与路由合同

### 3.1 前端页面

| 路由 | 页面 | v0.0.1 要求 |
| --- | --- | --- |
| `/` | 首页 | 展示产品信息和短链创建区域。未授权时置灰并提示。 |
| `/setup` | 初始化页 | 未初始化时可访问，用于首次初始化。 |
| `/login` | 登录页 | 本地账号登录。 |
| `/link` | 我的短链 | 展示当前用户自己的短链，支持复制、打开、启用或禁用、软删除。 |
| `/admin/link` | 全站短链管理 | 管理员查看和管理全站短链。 |
| `/admin/user/new` | 创建用户 | 管理员创建普通用户的最小入口。 |

### 3.2 后端路由

| 路由 | 说明 |
| --- | --- |
| `/api/v1/*` | 后端 API。 |
| `/{slug}` | 短链访问路由，优先级低于固定路由和 API 路由。 |

固定路由必须优先于短链短码路由。

### 3.3 保留路径和短码约束

v0.0.1 的短码创建必须按首段路径检查保留路径。以下首段路径不得作为短码写入：

```text
api
assets
setup
login
link
links
admin
```

说明：

- `/api/v1/*` 和 `/assets/*` 始终优先于短码路由。
- `/setup`、`/login`、`/link`、`/admin/*` 是固定前端路由。
- `links` 是旧页面路径首段，继续作为保留短码，避免升级后与历史固定路由冲突。
- `/` 是首页，不参与短码创建，但必须优先于短码路由。
- 后续新增固定前端路由或静态资源前缀时，必须同步更新本清单。

## 4. 数据库 schema 合同

### 4.1 通用约定

- 默认主键使用 `uuid`，`system_setting.key` 和 `session.id` 等明确声明的字段例外。
- 时间字段使用 `timestamptz`。
- 软删除字段统一使用 `deleted_at timestamptz null`。
- JSON 配置使用 `jsonb`。
- 所有表使用单数蛇形命名。
- 用户表使用 `app_user`，避免与数据库内置用户概念混淆。

### 4.2 `system_setting`

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `key` | `text` | 主键 | 设置键。 |
| `value` | `jsonb` | 非空 | 设置值。 |
| `created_at` | `timestamptz` | 非空 | 创建时间。 |
| `updated_at` | `timestamptz` | 非空 | 更新时间。 |

必须包含设置：

- `site.name`
- `site.initialized`
- `site.system_domain`
- `site.default_short_link_domain`
- `site.default_language`
- `site.default_theme`

### 4.3 `user_group`

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `uuid` | 主键 | 用户组 ID。 |

| `name` | `text` | 非空 | 展示名称。 |
| `description` | `text` | 非空，默认空字符串 | 描述。 |
| `permissions` | `jsonb` | 非空 | 权限字符串数组。 |
| `builtin` | `boolean` | 非空 | 是否内置。 |
| `created_at` | `timestamptz` | 非空 | 创建时间。 |
| `updated_at` | `timestamptz` | 非空 | 更新时间。 |

### 4.4 `app_user`

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |

| `username` | `text` | 非空，唯一 | 本地登录用户名。 |

| `created_at` | `timestamptz` | 非空 | 创建时间。 |

| `deleted_at` | `timestamptz` | 可空 | 预留软删除。v0.0.1 不提供用户删除 UI。 |

### 4.5 `session`

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `text` | 主键 | 服务端会话 ID，使用加密随机生成的 opaque token。 |
| `user_id` | `uuid` | 非空，外键 | 登录用户。 |
| `expires_at` | `timestamptz` | 非空，索引 | 过期时间。 |
| `last_seen_at` | `timestamptz` | 非空 | 最近访问时间。 |
| `revoked_at` | `timestamptz` | 可空 | 撤销时间。 |
| `created_at` | `timestamptz` | 非空 | 创建时间。 |

### 4.6 `domain`

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `uuid` | 主键 | 域名 ID。 |
| `host` | `text` | 非空，唯一 | 域名。 |
| `display_name` | `text` | 非空 | 展示名。 |
| `purpose` | `text` | 非空 | `system`、`short_link` 或 `both`。 |
| `enabled` | `boolean` | 非空 | 是否启用。 |
| `is_default` | `boolean` | 非空 | 是否全局默认短链访问域名。 |
| `created_at` | `timestamptz` | 非空 | 创建时间。 |
| `updated_at` | `timestamptz` | 非空 | 更新时间。 |

v0.0.1 只允许一个启用的默认短链访问域名。

### 4.7 `short_link`

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `uuid` | 主键 | 短链 ID。 |
| `owner_id` | `uuid` | 非空，外键 | 创建者。 |
| `domain_id` | `uuid` | 非空，外键 | 短链访问域名。 |
| `slug` | `text` | 非空，唯一 | 全系统唯一短码，保存小写字母数字。 |
| `target_url` | `text` | 非空 | 目标 URL。 |
| `status` | `text` | 非空 | `active` 或 `disabled`。 |
| `created_at` | `timestamptz` | 非空 | 创建时间。 |
| `updated_at` | `timestamptz` | 非空 | 更新时间。 |
| `deleted_at` | `timestamptz` | 可空 | 软删除时间。 |

短码全系统唯一。软删除后短码不释放，不自动复用。

短码规则：

- v0.0.1 默认生成 6 位随机短码。
- 短码只允许小写字母和数字：`[a-z0-9]`。
- 系统生成、保存和展示的短码都使用小写。
- 如果后续开放用户自定义短码，输入应先归一化为小写再校验和保存。
- 短码唯一性、保留路径校验和访问查找都基于归一化后的小写值。

## 5. API 合同

API 使用语义化路径，业务操作只使用 `GET` 和 `POST` 方法，不使用 `PUT`、`PATCH`、`DELETE` 表达业务动作。

统一响应：

```json
{
  "code": 0,
  "message": "OK",
  "data": {},
  "meta": {}
}
```

业务失败响应：

```json
{
  "code": 200103,
  "message": "目标链接不合法",
  "data": null,
  "meta": {}
}
```

HTTP 状态码规则：

- 应用层正常处理的业务成功和业务失败统一返回 `200 OK`。
- 未登录访问者默认映射为内置 `guest` 身份。对允许匿名进入应用层的接口，`guest` 权限不足属于业务无权限，返回 `200 OK`，响应体 `code` 为 `120001`。
- `401 Unauthorized` 仅用于需要有效登录态但未提供、Cookie 无效或会话失效的场景，用于前端自动跳转登录。
- `403 Forbidden` 仅用于已识别为登录用户，但登录态有效且基础访问控制失败的场景。普通业务权限失败优先使用统一业务 `code`。
- 未进入应用层的基础设施错误可以返回 `500 Internal Server Error`，但不作为业务错误表达方式。

错误码分段：

| 范围 | 含义 |
| --- | --- |
| `0` | 成功。 |
| `100000`-`109999` | 通用请求、参数和校验错误。 |
| `110000`-`119999` | 认证和会话错误。 |
| `120000`-`129999` | 权限错误。 |
| `200000`-`299999` | 短链错误。 |
| `300000`-`399999` | 用户错误。 |
| `400000`-`499999` | 域名错误。 |
| `900000`-`999999` | 系统和初始化错误。 |

v0.0.1 预定义错误码：

| code | 说明 |
| --- | --- |
| `0` | 成功。 |
| `100001` | 请求字段不合法。 |
| `110101` | 用户名或密码错误。 |
| `110102` | 用户已禁用。 |
| `120001` | 无权限执行操作。 |
| `200101` | 短码冲突。 |
| `200102` | 短码命中保留路径。 |
| `200103` | 目标链接不合法。 |
| `200104` | 短链不存在。 |
| `200105` | 短链已禁用。 |
| `300101` | 用户名已存在。 |
| `900101` | 系统已初始化。 |

```text

```


- 提交 `targetUrl` 时必须重新执行目标链接安全校验。

- 当前用户更新不存在、已软删除或不属于自己的短链时，统一返回 `200 OK`，响应体 `code` 为 `200104`，避免泄露他人短链是否存在。

| --- | --- |
| `guest` | 无实际操作权限。 |
| `user` | `short_link:create`、`short_link:read_own`、`short_link:update_own`、`short_link:delete_own`、`domain:use_default`。 |
| `admin` | v0.0.1 全部权限。 |

### 6.3 内置用户

系统必须创建内置 `guest` 用户：

- 不允许登录。
- 不允许删除。
- 默认属于 `guest` 用户组。
- 用于表示未登录访问者。

`guest` 权限不足时按业务无权限处理，返回 `200 OK` 和 `120001`，不得用 `401` 或 `403` 表达。

## 7. Cookie Session 安全合同

- Cookie 名称：`moeurl_session`。
- Cookie 必须设置 `HttpOnly`。
- Cookie `SameSite` 使用 `Lax`。
- 生产环境必须设置 `Secure`。
- Cookie Path 使用 `/`。
- 登录成功后生成新的加密随机 session ID。
- 退出登录必须撤销服务端 session 并清理 Cookie。
- 每次授权操作必须重新检查用户状态和权限。
- 用户被禁用后，不得继续执行授权操作。
- v0.0.1 不要求单独实现 CSRF Token；默认依赖 `SameSite=Lax` 和 JSON API 边界，后续如开放跨站嵌入或第三方表单再补充 CSRF 机制。

## 8. 事件预留合同

后端应定义统一事件记录接口，例如：

```go
type Recorder interface {
    Record(ctx context.Context, event Event) error
}
```

v0.0.1 可以使用 no-op 实现或结构化日志实现。短链访问流程必须在以下路径调用事件接口：

| 事件 | 调用点 |
| --- | --- |
| `short_link_opened` | 成功识别域名和短码后。 |
| `access_condition_checked` | 完成状态和访问条件检查后。 |
| `redirect_blocked` | 短链因不存在、软删除、禁用或访问条件被阻止时。 |
| `redirect_initiated` | 准备返回跳转响应前。 |
| `redirect_response_sent` | 跳转响应构造完成后。 |
