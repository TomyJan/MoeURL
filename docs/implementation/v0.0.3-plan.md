# v0.0.3 实施计划

## 1. 目标

v0.0.3 是 MoeURL 的第一轮 UI/UX 精细化重设计版本，目标是在 v0.0.2 已完成的功能闭环之上，建立稳定的前端视觉系统、首页体验、控制台框架和短链生成组件复用模式。

本版本的主线是：

```text
设计基线同步 → 主题 token 和全局视觉 → 可复用短链生成 → 首页重设计 → 控制台框架迁移 → 响应式和验收收尾
```

v0.0.3 不实现 v0.1.0 的统计、中间页跳转、访问密码、二维码和短链过期时间。实现过程中如发现现有前端结构不便迁移，可以做服务于本阶段目标的小范围拆分，但不得借机重写后端业务模型。

## 3. 技术边界

v0.0.3 继续遵循现有技术栈：

- 前端：Vue 3、Vite、TypeScript、Vuetify 4。
- 状态：Pinia、TanStack Query for Vue。
- 国际化：vue-i18n。
- 表单校验：vee-validate、zod。
- 测试：Vitest、Playwright。
- 后端：Go、Chi、SQLC、Goose、PostgreSQL。

本版本优先修改前端布局、组件、样式、测试和少量文档。原则上不新增后端 API。短链创建、登录态、权限判断、用户管理和短链列表继续使用 v0.0.2 既有契约。

## 4. 阶段划分

v0.0.3 分为 6 个阶段：

1. 文档和设计基线同步。
2. 主题 token 与全局视觉系统。
3. 短链生成组件复用。
4. 首页重设计。
5. 控制台 Shell 和页面迁移。
6. 响应式、测试、E2E 和验收收尾。

每个阶段都应产出可运行、可验证的结果，并优先保持小提交。

## 5. 阶段 1：文档和设计基线同步

### 目标

让项目文档明确 v0.0.3 是当前优先目标，并把此前确认的 UI/UX 决策沉淀为开发依据。

### 实施内容

- 新增 v0.0.3 产品范围。
- 新增 v0.0.3 实施计划、任务清单和验收清单。
- 更新文档中心阅读顺序。
- 更新产品总览当前优先目标。
- 更新路线图，在 v0.0.2 和 v0.1.0 之间插入 v0.0.3。
- 更新实施文档索引。

### 验收

- 开发者可以从 `docs/README.md` 找到 v0.0.3 的范围、计划、任务和验收清单。
- 文档明确 v0.0.3 不包含统计、访问密码、二维码和短链过期时间。
- 文档引用已确认的首页、控制台和短链生成结构设计，并明确当前配色与交互基线为「雾蓝石墨」。

## 6. 阶段 2：主题 token 与全局视觉系统

### 目标

替换临时 UI 配色，建立 v0.0.3 的浅色、深色和圆角视觉基线。

### 实施内容

- 在 Vuetify theme 中定义新的浅色主题 token。
- 在 Vuetify theme 中定义新的深色主题 token。
- 保留现有主题偏好读取、保存、自动模式和切换能力。
- 抽出或整理全局 CSS 变量，用于页面背景、表面色、边框、阴影和圆角。
- 建立常用组件的大圆角覆盖规则，包括按钮、输入框、弹窗、抽屉、表格容器和页面面板。
- 调整基础文字颜色、辅助文字颜色、分隔线和状态标签样式。

### 验收

- 浅色模式视觉符合「雾蓝石墨」方向下的冷白、雾蓝和石墨文字体系。
- 深色模式视觉符合「雾蓝石墨」方向下的蓝黑、钢蓝和冰蓝体系。
- 首页、登录页、控制台页面在浅色和深色模式下没有明显色块割裂。
- 主题切换测试继续通过。

## 7. 阶段 3：短链生成组件复用

### 目标

将短链创建能力抽象为可复用组件，同时服务首页和控制台。

### 实施内容

- 抽出 `ShortLinkCreatePanel` 或等价组件。
- 组件复用现有短链创建 API、校验、成功展示和复制逻辑。
- 支持首页 inline 展示。
- 支持控制台 modal 展示。
- 支持移动端 bottom-sheet 展示。
- 保留权限不足、未登录、创建失败、复制成功和加载状态。
- 不在本阶段暴露访问密码、过期时间、二维码和中间页跳转。

### 验收

- 首页和控制台创建短链使用同一套核心组件。
- 创建成功后能展示短链并复制。
- 无权限时组件保留结构并给出明确提示。
- 组件测试覆盖成功、失败、无权限和复制状态。

## 8. 阶段 4：首页重设计

### 目标

让首页从临时页面变成 MoeURL 的核心体验入口，首屏聚焦短链快速生成，同时保留轻量介绍。

### 实施内容

- 新建或重构首页布局组件。
- 首页顶部导航视觉融入背景，不使用独立背景的导航栏。
- 左侧展示 Logo 和站点名称。
- 右侧根据登录态展示「登录」或「头像 + 昵称」。
- 已登录头像昵称点击进入控制台。
- 首屏占满主要视口，居中放置标题、简短说明和短链生成组件。
- 首屏底部露出下一段内容或滚动提示。
- 下翻区域展示权限可控、自托管友好、快速管理和现代体验等轻量介绍。
- 桌面和移动端都保持首屏重点清晰。

### 验收

- 未登录用户首页右侧显示登录入口。
- 已登录用户首页右侧显示头像和昵称，并可进入控制台。
- 首页首屏在桌面和移动端都以短链生成组件为中心。
- 下翻介绍不影响首屏生成短链。
- 首页测试覆盖登录态入口和生成组件渲染。

## 9. 阶段 5：控制台 Shell 和页面迁移

### 目标

建立统一控制台框架，并迁移 v0.0.2 已有页面，消除每个页面独立拼装布局的问题。

### 实施内容

- 新建 `ConsoleShell` 或等价控制台框架。
- 桌面端采用左侧导航 + 主内容区。
- 移动端采用顶部栏 + 汉堡按钮 + 抽屉导航。
- 左侧导航顶部展示品牌区。
- 左侧导航提供「新建短链」主动作。
- 桌面端账号入口放在左侧导航底部。
- 移动端顶部右侧保留头像入口。
- 主内容区右上角只放页面级动作。
- 根据权限展示我的短链、全站短链和用户管理入口；创建用户作为用户管理页内页面级动作，不作为独立主导航入口。
- 迁移我的短链页、全站短链页、用户管理页和创建用户页。
- 可以展示概览、统计分析、用户组与权限、系统设置等规划中占位入口；导航项应展示规划中徽标，占位页必须明确标记为规划中，不展示假数据，不开放未实现表单或业务动作。

### 验收

- 管理员看到我的短链、全站短链、用户管理入口，并可在用户管理页通过页面级动作进入创建用户。
- 管理员可以看到带规划中徽标的占位入口，但不会误认为统计分析、用户组与权限、系统设置已经实现。
- 普通用户只看到自己有权限访问的入口。
- 桌面端账号入口位于左侧导航底部。
- 移动端汉堡菜单可以打开控制台导航。
- 控制台「新建短链」可以打开生成浮窗。
- 既有短链筛选、搜索、用户管理操作不回退。

## 10. 阶段 6：响应式、测试、E2E 和验收收尾

### 目标

确认 v0.0.3 的视觉重构在主要设备、主题和权限状态下稳定，并确保 v0.0.2 功能不回退。

### 实施内容

- 补齐浅色和深色模式下的组件测试。
- 补齐首页登录态、控制台导航、短链生成组件和权限入口测试。
- 补齐 Playwright E2E，覆盖首页创建、登录进入控制台、控制台创建短链和管理员页面访问。
- 检查移动端视口下导航、浮窗、列表和表单不会溢出。
- 运行前端完整质量门禁。
- 运行后端基础回归命令，确认无后端回归。
- 更新 v0.0.3 验收清单。

### 验收

- `cd web && pnpm lint` 通过。
- `cd web && pnpm typecheck` 通过。
- `cd web && pnpm test` 通过。
- `cd web && pnpm test:coverage` 通过。
- `cd web && pnpm build` 通过。
- `cd web && pnpm test:e2e` 通过。
- `go test ./...` 通过。
- `go vet ./...` 通过。
- v0.0.3 验收清单记录自动化和必要手工验证结果。

## 11. 推荐提交拆分

建议按以下顺序提交：

```text
docs(planning): add ui ux refinement scope
style(web): add refreshed moeurl themes
feat(web): reuse short link create panel
feat(web): redesign home experience
feat(web): add console shell layout
test(web): cover ui refinement flows
docs(acceptance): record ui refinement verification
```

每次提交前运行与改动相关的最小验证。发布前运行完整验证。
