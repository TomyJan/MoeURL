# v0.1.0 实施计划

## 1. 目标

v0.1.0 将 v0.0.4 的基础访问统计扩展为可用的单短链分析页，不实现短链控制类功能。

## 3. 阶段

1. 同步产品、规格、技术基线和验收边界。
2. 通过 Goose migration 扩展事件表，并以 SQLC 提供趋势和维度聚合。
3. 在成功跳转响应后采集经过最小化处理的请求维度。
4. 提供用户与管理员统计 API，并覆盖权限、无效输入和基础设施失败。
5. 交付控制台分析页、列表入口和双语文案。
6. 在 CI 执行 E2E，记录完整验收结果。

## 4. 技术要求

- 继续使用 Go、Chi、SQLC、Goose、PostgreSQL、Vue 3、Vuetify 4 与 TanStack Query。
- 图表使用 `chart.js`，组件卸载时销毁实例。
- 事件写入保持有界、异步、短超时和可丢弃；不得改变已发送跳转响应。
- 所有统计查询只统计 `redirect_response_sent`，并在服务层验证短链可见性。

## 5. 验收重点

- 连续 7 天趋势与零值日期正确。
- 维度统计不泄露原始请求数据。
- 普通用户不能读取其他用户统计，管理员可以读取全站统计。
- 无数据、加载、业务错误和移动端均可用。
- E2E 作为发布门禁实际运行，不以未启动替代通过。
