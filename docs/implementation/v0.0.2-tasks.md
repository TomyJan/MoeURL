# v0.0.2 任务级实施清单

> 本文档把 [v0.0.2 实施计划](./v0.0.2-plan.md) 拆成可执行任务。具体编码时，应继续遵循技术基线、v0.0.1 工程合同和覆盖率门禁。

## 1. 当前状态

v0.0.1 基础可用闭环已经完成并合并。当前分支从 `master` 拉出，用于规划和实施 v0.0.2。

v0.0.2 的优先级顺序是：

1. 文档和验收记录同步。
2. 管理员用户管理最小闭环。
3. 短链列表筛选和搜索体验。
4. 测试、E2E 和部署验证补强。

## 3. 阶段 2：管理员用户管理最小闭环

### 2.1 后端用户列表 API

建议文件：

- 修改：`queries/user.sql`
- 修改：`internal/user/service.go`
- 修改：`internal/user/handler.go`
- 修改：`internal/http/router.go`
- 测试：`internal/user/service_test.go`

验收：

- `GET /api/v1/admin/user/list` 返回未软删除用户。
- 普通用户调用返回业务无权限错误。
- 列表包含账号、昵称、用户组、状态、是否内置、创建时间和更新时间。

### 2.2 后端用户更新 API

建议文件：

- 修改：`queries/user.sql`
- 修改：`internal/user/service.go`
- 修改：`internal/user/handler.go`
- 测试：`internal/user/service_test.go`

验收：

- `POST /api/v1/admin/user/update` 可以修改普通用户昵称和状态。
- 修改内置 `guest` 用户被拒绝。
- 禁用用户后，该用户不能继续执行授权操作。

### 2.3 后端重置密码 API

建议文件：

- 修改：`internal/user/service.go`
- 修改：`internal/user/handler.go`
- 修改：`internal/auth/password.go`
- 测试：`internal/user/service_test.go`

验收：

- `POST /api/v1/admin/user/reset-password` 可以重置普通用户密码。
- 重置后旧密码不能登录，新密码可以登录。
- 内置 `guest` 用户不可重置密码。

### 2.4 前端用户管理页面

建议文件：

- 创建：`web/src/pages/AdminUsersPage.vue`
- 创建或修改：`web/src/entities/user/api.ts`
- 创建或修改：`web/src/entities/user/model.ts`
- 修改：`web/src/app/router.ts`
- 修改：`web/src/widgets/app-shell/AppShell.vue`

验收：

- 管理员可以访问 `/admin/user`。
- 页面展示用户列表和状态。
- 页面支持启用、禁用、修改昵称和重置密码。
- 普通用户访问时显示无权限或被路由守卫拦截。

## 4. 阶段 3：短链列表筛选和搜索体验

### 3.1 扩展短链列表查询

建议文件：

- 修改：`queries/short_link.sql`
- 修改：`internal/shortlink/service.go`
- 修改：`internal/shortlink/handler.go`
- 测试：`internal/shortlink/service_test.go`

验收：

- 我的短链列表支持 `status` 参数。

- 普通用户不能通过筛选看到他人短链。

### 3.2 前端我的短链筛选

建议文件：

- 修改：`web/src/pages/MyLinksPage.vue`

验收：

- 我的短链页支持全部、启用、禁用筛选。
- 空结果显示清晰提示。
- 筛选条件变化时刷新服务端查询。

### 3.3 前端管理员短链搜索

建议文件：
