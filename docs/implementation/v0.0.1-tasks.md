# v0.0.1 任务级实施清单

> 本文档用于把 [v0.0.1 实施规划](v0.0.1-plan.md) 进一步拆成可执行任务。具体代码实现时，应结合实际技术栈把每个任务转换为更细的开发步骤、测试用例和提交。

## 1. 执行方式

建议按阶段顺序执行。每个任务完成后，应至少满足：

- 相关功能可以本地运行或通过测试验证。
- 对应文档没有明显过期描述。
- 变更可以独立提交。

任务状态建议使用：

```text
未开始 → 进行中 → 已实现 → 已验证 → 已提交
```

## 2. 阶段 1：项目技术基线与应用骨架

### 1.1 确认技术栈和开发命令

目标：明确项目采用的前端、后端、数据库、构建和测试方案。

交付：

- 项目可以安装依赖。
- 项目可以本地启动。
- 项目可以运行测试。
- README 或实施文档记录常用命令。

验收：

- `dev` 命令可启动本地应用。
- `build` 命令可完成构建。
- `test` 命令可运行测试。

### 1.2 建立基础路由

目标：建立 v0.0.1 所需的最小页面路由。

页面：

- 首页。
- 登录页。
- 初始化页。
- 我的短链页。
- 基础管理入口。
- 短链访问路由。
- 404 页面。

验收：

- 每个固定路由都可以访问。
- 未知路径进入 404 页面。
- 固定路由优先级高于短码路由。

### 1.3 建立基础布局

目标：提供统一页面框架。

内容：

- 顶部导航。
- 主要内容区域。
- 基础页脚。
- 登录状态入口。
- 语言切换入口。
- 主题切换入口。

验收：

- 首页、登录页、我的短链页使用统一布局。
- 移动端基础布局不破坏主流程。

### 1.4 建立错误状态页面

目标：为系统错误和短链访问状态提供基础展示能力。

页面：

- 404 页面。
- 短链不存在页面。
- 短链已禁用页面。
- 系统未初始化提示或跳转。

验收：

- 访问不存在短码时有明确提示。
- 访问禁用短链时不会跳转目标 URL。

## 3. 阶段 2：数据模型与基础领域对象

### 2.1 定义用户模型

字段至少包括：

- 用户 ID。
- 账号名。
- 密码哈希。
- 昵称。
- 状态。
- 主用户组 ID。
- 创建时间。
- 更新时间。

验收：

- 可以创建管理员用户。
- 可以创建普通用户。
- 可以禁用用户。

### 2.2 定义用户组和权限模型

用户组字段至少包括：

- 用户组 ID。
- 名称。
- 标识。
- 描述。
- 是否内置。
- 权限集合。
- 创建时间。
- 更新时间。

内置用户组：

- `guest`
- `user`
- `admin`

验收：

- 系统可以初始化内置用户组。
- 内置用户组不能被删除。
- 用户权限可以通过主用户组计算。

### 2.3 定义域名模型

字段至少包括：

- 域名 ID。
- 域名。
- 用途。
- 是否启用。
- 是否默认。
- 创建时间。
- 更新时间。

验收：

- 系统可以保存默认短链访问域名。
- 创建短链时可以引用默认短链访问域名。

### 2.4 定义短链模型

字段至少包括：

- 短链 ID。
- 短码。
- 目标 URL。
- 所属用户 ID。
- 域名 ID 或域名快照。
- 跳转方式。
- 状态。
- 审核状态或审核扩展字段。
- 创建时间。
- 更新时间。
- 删除时间。

验收：

- 短码全系统唯一。
- 短链可以关联用户和域名。
- 软删除后短码仍不可复用。

### 2.5 定义系统设置模型

字段至少包括：

- 是否已初始化。
- 站点名称。
- 默认语言。
- 默认主题。
- 默认短链访问域名。
- 默认短码长度。

验收：

- 系统可以判断是否已初始化。
- 初始化流程可以写入系统设置。

## 4. 阶段 3：首次初始化流程

### 3.1 实现初始化状态检查

目标：根据系统设置判断是否进入初始化模式。

验收：

- 未初始化时访问首页会进入初始化流程。
- 未初始化时登录和管理后台不可用。
- 已初始化后不能重复进入初始化流程。

### 3.2 实现初始化表单

表单至少包含：

- 管理员账号。
- 管理员密码。
- 管理员昵称。
- 站点名称。
- 默认短链访问域名。
- 默认语言。
- 默认主题。

验收：

- 表单校验必填字段。
- 密码符合基础强度要求。
- 默认短链访问域名格式合法。

### 3.3 写入初始化结果

目标：创建第一个管理员，并写入系统设置。

验收：

- 第一个用户属于 `admin` 用户组。
- 系统设置标记为已初始化。
- 初始化完成后可以登录。

## 5. 阶段 4：本地用户、登录与会话

### 4.1 实现密码哈希和校验

目标：本地账号密码不保存明文。

验收：

- 创建用户时保存密码哈希。
- 登录时通过哈希校验密码。
- 错误密码不能登录。

### 4.2 实现登录页

登录页至少包含：

- 账号输入框。
- 密码输入框。
- 登录按钮。
- 错误提示。
- 语言切换。
- 主题切换。

验收：

- 管理员可以登录。
- 普通用户可以登录。
- 登录失败展示明确错误。

### 4.3 实现会话和当前用户读取

目标：登录后可以识别当前用户，未登录时识别为 `guest`。

验收：

- 登录后页面显示当前用户入口。
- 未登录访问者按 `guest` 身份计算权限。
- 退出登录后回到未登录状态。

### 4.4 实现用户状态检查

目标：禁用用户不能登录或继续操作。

验收：

- 禁用用户登录失败。
- 已登录用户被禁用后无法继续执行需要授权的操作。

## 6. 阶段 5：权限与内置用户组

### 5.1 初始化 v0.0.1 最小权限

权限包括：

- `short_link:create`
- `short_link:read_own`
- `short_link:update_own`
- `short_link:delete_own`
- `short_link:manage_all`
- `domain:use_default`
- `admin:access`

验收：

- 权限标识稳定。
- 权限可以分配给用户组。

### 5.2 配置默认用户组权限

默认权限：

- `guest`：无实际操作权限。
- `user`：可创建和管理自己的短链。
- `admin`：拥有 v0.0.1 全部权限。

验收：

- `guest` 不能创建短链。
- `user` 可以创建短链。
- `admin` 可以进入管理入口。

### 5.3 实现权限判断服务

目标：提供统一权限判断入口。

验收：

- 页面展示和后端操作使用同一套权限结果。
- 无权限操作被拒绝。
- 不依赖前端隐藏作为唯一保护。

### 5.4 接入首页创建权限提示

目标：首页创建区域根据权限展示可用或置灰状态。

验收：

- `guest` 看到创建区域置灰和登录提示。
- `user` 和 `admin` 可以使用创建区域。

## 7. 阶段 6：短链创建与管理

### 6.1 实现目标 URL 校验

规则：

- URL 格式合法。
- 只允许 `http://` 和 `https://`。
- 禁止 `javascript:`、`data:`、`file:` 等协议。
- 创建和编辑时都校验。

验收：

- 合法 HTTP / HTTPS URL 可以通过。
- 非法 URL 被拒绝。
- 禁止协议被拒绝。

### 6.2 实现短码生成和唯一性校验

目标：生成随机短码，并确保全系统唯一。

验收：

- 创建短链时自动生成短码。
- 重复短码不会写入成功。
- 与固定路由冲突的短码不会写入成功。

### 6.3 实现首页创建短链

目标：用户可以在首页创建短链。

验收：

- 有权限用户可以创建短链。
- 创建成功后展示短链地址。
- 创建成功后提供复制、打开和继续创建入口。

### 6.4 实现我的短链列表

列表至少展示：

- 短链地址。
- 目标 URL。
- 状态。
- 创建时间。

验收：

- 用户可以看到自己的短链。
- 用户看不到他人的短链。
- 空列表有清晰提示。

### 6.5 实现短链状态操作

操作包括：

- 启用。
- 禁用。
- 软删除。

验收：

- 用户可以禁用自己的短链。
- 用户可以重新启用自己的短链。
- 用户可以软删除自己的短链。
- 软删除后普通列表不展示。

## 8. 阶段 7：短链访问与直接跳转

### 7.1 实现短链访问路由

目标：固定路由优先，短码路由其次。

验收：

- 固定系统路由不会被短码覆盖。
- 不存在短码进入 404 或短链不存在页。

### 7.2 实现短链查找

目标：根据访问域名和短码查找短链。

验收：

- 有效短码可以找到对应短链。
- 已删除短链按不存在处理。

### 7.3 实现状态检查

v0.0.1 至少处理：

- 不存在。
- 已删除。
- 已禁用。
- 正常启用。

验收：

- 已禁用短链不会跳转目标 URL。
- 已删除短链按不存在处理。
- 正常短链可以继续跳转。

### 7.4 实现直接跳转

目标：正常短链返回跳转响应。

验收：

- 访问短链会跳转到目标 URL。
- 目标 URL 保持创建时的合法 URL。

### 7.5 预留访问事件触发点

事件包括：

- `short_link_opened`
- `access_condition_checked`
- `redirect_initiated`
- `redirect_response_sent`
- `redirect_blocked`

验收：

- 访问流程中有统一事件触发位置。
- v0.0.1 不要求实现完整统计展示。

## 9. 阶段 8：管理员全站短链入口

### 8.1 实现基础管理入口

目标：只有具备 `admin:access` 的用户可以进入。

验收：

- 非管理员不能访问管理入口。
- 管理员可以访问管理入口。

### 8.2 实现全站短链列表

列表至少展示：

- 短链地址。
- 目标 URL。
- 所属用户。
- 状态。
- 创建时间。

验收：

- 管理员可以看到全部短链。
- 普通用户不能访问全站短链列表。

### 8.3 实现管理员短链操作

操作包括：

- 启用任意短链。
- 禁用任意短链。
- 软删除任意短链。

验收：

- 管理员可以管理全站短链。
- 操作受 `short_link:manage_all` 权限保护。

## 10. 阶段 9：多语言、主题和基础体验收尾

### 9.1 实现多语言基础架构

至少支持：

- 简体中文。
- 英语。

覆盖页面：

- 首页。
- 登录页。
- 我的短链页。
- 基础管理入口。
- 短链访问状态页。
- 常见错误提示。

验收：

- 用户可以切换简体中文和英语。
- 关键页面文案随语言切换。

### 9.2 实现主题模式基础架构

至少支持：

- 跟随系统。
- 浅色模式。
- 深色模式。

验收：

- 用户可以切换主题模式。
- 主题选择在刷新后保持，或按当前设计的存储策略保持。

### 9.3 完成响应式基础适配

重点页面：

- 首页。
- 登录页。
- 我的短链页。
- 短链状态页。

验收：

- 移动端可以完成登录、创建短链、复制短链和查看短链列表。

### 9.4 补齐基础状态体验

状态包括：

- 加载中。
- 空状态。
- 错误状态。
- 无权限状态。
- 操作成功提示。

验收：

- 核心流程不会出现无说明的空白页。
- 用户能理解操作失败原因。

## 11. 推荐提交拆分

建议每个阶段至少一个提交，复杂阶段可拆成多个提交：

```text
chore(项目): 初始化应用骨架
feat(模型): 添加 v0.0.1 基础领域模型
feat(初始化): 添加首次初始化流程
feat(认证): 添加本地登录与会话
feat(权限): 添加内置用户组和最小权限判断
feat(短链): 添加短链创建与管理
feat(跳转): 添加短链访问与直接跳转
feat(管理): 添加全站短链管理入口
feat(体验): 添加多语言和主题基础能力
```

提交前应运行对应测试和基础验证命令。
