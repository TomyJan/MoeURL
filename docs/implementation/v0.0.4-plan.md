# v0.0.4 实施计划

## 1. 目标

v0.0.4 是 MoeURL 的基础统计前置版本，目标是在 v0.0.3 已完成的控制台体验之上，建立短链访问事件持久化和列表级基础统计展示。

本版本主线是：

```text
范围文档同步 → 访问事件表 → 事件记录器 → 列表统计聚合 → 控制台展示 → 验收收尾
```

v0.0.4 不实现完整统计分析页、访问明细、来源/设备/地区维度、导出、访问密码、过期时间、二维码和跳转模式。

## 3. 技术边界

v0.0.4 继续遵循现有技术栈：

- 后端：Go、Chi、SQLC、Goose、PostgreSQL。
- 前端：Vue 3、Vite、TypeScript、Vuetify 4。
- 状态：TanStack Query for Vue。
- 测试：go test、后端覆盖率门禁、Vitest、Playwright。

本版本允许新增数据库 migration、SQLC 查询、事件记录器实现、短链列表响应字段和前端列表展示。原则上不新增独立统计页，不改变既有 API 路径和业务动作风格。

## 4. 阶段划分

v0.0.4 分为 5 个阶段：

1. 文档和范围同步。
2. 数据库事件表和 SQLC 查询。
3. 访问事件记录和后端统计响应。
4. 前端模型和列表展示。
5. 测试、E2E 和验收收尾。

每个阶段都应产出可验证结果，并遵循先测试、后实现的 TDD 节奏。

让项目文档明确 v0.0.4 是当前优先目标，并说明基础统计前置的边界。

- 新增 v0.0.4 实施计划、任务清单和验收清单。
- 更新文档中心阅读顺序。

### 验收

- 开发者可以从 `docs/README.md` 找到 v0.0.4 的范围、计划、任务和验收清单。

- 文档明确 v0.0.4 不包含完整统计页、访问明细、维度统计和导出。

## 6. 阶段 2：数据库事件表和 SQLC 查询

### 目标

建立可持久化短链访问事件和聚合基础统计的数据基础。

### 实施内容

- 新增 Goose migration，创建 `short_link_event` 表。
- 表字段包含 `id`、`short_link_id`、`event_type`、`created_at`。
- 为 `short_link_id`、`event_type` 和 `created_at` 建立必要索引。
- 新增 SQLC 查询：
  - 创建短链事件。
  - 按短链聚合总访问量、今日访问量和最近访问时间。
  - 列表查询通过聚合子查询返回基础统计。
- 重新生成 SQLC Go 代码。

### 验收

- migration 测试通过。
- SQLC 生成代码和查询测试通过。
- 无访问事件时统计字段为 `0` 和空最近访问时间。

## 7. 阶段 3：访问事件记录和后端统计响应

### 目标

让成功短链跳转写入事件，并让列表 API 返回基础统计。

### 实施内容

- 在 `internal/event` 中新增数据库 recorder。
- `RedirectService` 继续通过 `event.Recorder` 记录事件。
- 成功跳转时写入 `redirect_response_sent` 事件。
- 短链不存在、软删除、禁用时不写入成功访问统计。
- 扩展短链服务模型，增加 `Stats` 字段。
- 我的短链和管理员全站短链列表返回基础统计。

### 验收

- 成功访问短链后事件表写入一条成功访问事件。
- 禁用短链访问不增加访问量。
- 我的短链列表返回基础统计。
- 管理员全站短链列表返回基础统计。
- 现有创建、更新、删除和筛选测试不回退。

## 8. 阶段 4：前端模型和列表展示

### 目标

在控制台短链列表中展示基础访问统计。

### 实施内容

- 扩展前端短链模型，增加 `stats`。
- API 测试覆盖统计字段解析。
- `ConsoleLinkList` 展示总访问量、今日访问量和最近访问时间。
- 无最近访问时间时显示「暂无访问」。
- i18n 补充中英文文案。

### 验收

- 我的短链列表能展示基础统计。
- 全站短链列表能展示基础统计。
- 无统计数据时显示稳定占位。
- 移动端列表不出现横向溢出。

## 9. 阶段 5：测试、E2E 和验收收尾

### 目标

确认 v0.0.4 不破坏 v0.0.3 和 v0.0.2 已完成能力。

### 实施内容

- 运行后端单元和集成测试。
- 运行后端覆盖率门禁。
- 运行前端 lint、类型检查、测试、覆盖率和构建。
- 必要时运行 E2E 主流程。
- 更新 v0.0.4 验收清单。

### 验收

- `go test ./...` 通过。
- `go vet ./...` 通过。
- 后端覆盖率门禁通过。
- `cd web && pnpm lint` 通过。

## 10. 推荐提交拆分

建议按以下顺序提交：

```text
docs(planning): add v0.0.4 statistics scope
feat(db): persist short link visit events
feat(api): expose short link basic statistics
feat(web): show basic link statistics
test: cover basic statistics flows
docs(acceptance): record v0.0.4 verification
```

每次提交前运行与改动相关的最小验证。发布前运行完整验证。
