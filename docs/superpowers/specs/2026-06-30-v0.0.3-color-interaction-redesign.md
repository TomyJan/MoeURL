# v0.0.3 配色与交互重设设计

## 1. 背景

当前 v0.0.3 UI/UX 重设计已经建立了首页、控制台 Shell、可复用短链生成组件、偏好切换组件和多级控制台导航。现有布局、圆角、密度和主要组件结构基本可继续使用，但配色和部分交互状态仍显得不稳定。

本设计用于替换现有绿色、薄荷和蜂蜜黄方向的主题配色，并重新定义按钮、输入框、菜单、状态标签和多级导航的交互语义。

设计预览文件：

- [v003-color-interaction-redesign.html](../prototypes/v003-color-interaction-redesign.html)

## 2. 设计目标

- 建立一套不参考现有绿色体系的新浅色和深色主题。
- 保留现有页面布局、大圆角、组件结构和控制台信息架构。
- 让首页保持清透产品感，同时让控制台更接近成熟工具界面。
- 明确主操作、焦点、选中、提醒、成功和错误的颜色职责。
- 让左侧多级导航在父级展开、子项选中、规划中占位和普通入口之间有清晰层级。
- 不新增业务能力，不改变后端 API 和前端数据流。

## 3. 非目标

- 不重写首页、控制台和短链生成组件的布局结构。
- 不改变 v0.0.3 已定义的页面范围和导航信息架构。
- 不引入新的 UI 组件库。
- 不实现统计、二维码、访问密码、过期时间、中间页跳转或系统设置表单。
- 不做主题后台配置。

## 4. 视觉主张

视觉方向为「雾蓝石墨」。

这套设计以冷静工具感作为骨架，以清透产品感作为表面。浅色主题使用冷白、雾蓝和石墨文字，深色主题使用蓝黑、钢蓝表面和冰蓝主操作。少量铜橙只用于提醒、规划中标识和当前位置 rail，不作为大面积按钮色。

### 4.1 内容计划

- 首页首屏继续围绕短链快速生成，不增加营销式卡片。
- 首页下翻区域继续作为轻量产品说明，不喧宾夺主。
- 控制台继续以左侧导航和右侧工作区为主体。
- 用户管理、我的短链、全站短链等页面沿用现有数据行结构。
- 规划中页面和规划中导航项只展示边界，不展示假数据。

### 4.2 交互主张

- 主操作按钮在 hover 时轻微上浮，并加深主色。
- 输入框聚焦时使用低饱和蓝色光晕，不使用高亮硬边框。
- 导航当前项使用低对比背景加暖色窄 rail，避免整块高饱和选中态。
- 父级菜单展开态和子项命中态分离：父级只表达展开，子项表达当前路由。
- 菜单、抽屉和弹窗保持短距离位移与透明度过渡，不使用夸张动效。

## 6. 组件状态规则

### 6.1 按钮

- 主按钮使用 `primary` 背景和反白文字。
- 主按钮 hover 使用更深或更亮的 `primary-hover`，并允许 `translateY(-1px)`。
- 次级按钮使用透明或表面色背景，边框使用 `outline`。
- 危险操作不使用大面积红色按钮，优先使用菜单项或确认弹窗中的危险语义。
- 按钮文字不使用负字距，不使用全大写。

### 6.2 输入框

- 默认背景使用 `surface-soft`，边框保持低对比。
- 聚焦状态使用 `primary` 派生的 4px 柔和光晕。
- 聚焦态不改变输入框尺寸，不造成布局跳动。
- 错误态使用错误色边框和短文案，不叠加主色光晕。

### 6.3 菜单和偏好切换

- 菜单背景使用 `surface`，边框使用 `outline`，阴影只表达浮层关系。
- 当前选项使用低透明度 `primary` 背景。
- 选项命中点使用小圆点或短标记，不使用大面积亮色块。
- 语言和主题菜单保持互斥展开。

### 6.4 状态标签

状态标签必须有明确语义：

- 信息：使用主色低透明背景。
- 启用 / 成功：使用成功色。
- 规划中 / 提醒：使用强调色。
- 禁用 / 错误：按业务语义分别使用强调色或错误色。

规划中标签只能用于未实现入口或占位页，不得用于真实业务状态。

## 7. 多级导航设计

### 7.1 桌面端左侧导航

控制台左侧导航继续按分组展示：

- Workspace：概览、我的短链、访问分析。
- Admin：全站短链、用户管理、系统设置。

其中 `用户管理` 是父级菜单。父级展开后包含：

- 用户。
- 用户组与权限。

### 7.2 父级展开态

父级展开态只表达「该分组已展开」，不直接表达当前页面。

视觉规则：

- 背景使用低透明度 `surface-strong`。
- 文字使用主文字或次级文字。
- caret 向下。
- 不显示暖色 rail，除非父级本身也是一个可访问页面且当前路由命中父级。

### 7.3 子项选中态

子项选中态表达当前路由。

视觉规则：

- 子项相对父级缩进。
- 子项列表左侧有细分隔线。
- 当前子项使用 `surface-strong` 背景。
- 当前子项文字使用 `primary`。
- 当前子项 rail 使用强调色。

### 7.4 规划中入口

规划中入口可以显示在导航中，但必须降低误导风险。

规则：

- 导航项后展示 `规划中` badge。
- badge 使用强调色低透明背景。
- 对应页面只展示能力边界说明，不展示假数据，不开放表单。
- 规划中入口仍可点击进入占位页，避免禁用项造成信息架构断裂。

### 7.5 移动端抽屉导航

移动端抽屉沿用同一层级语义：

- 父级展开态、子项缩进和规划中 badge 与桌面一致。
- 抽屉背景使用深色或浅色对应的 `surface`。
- 当前子项 rail 和文字必须在窄屏下仍清晰。
- 点击任意可访问子项后关闭抽屉。

## 8. 页面应用规则

### 8.1 首页

- 首页导航继续融入背景，不新增独立导航栏底色。
- 首页品牌、登录入口、偏好切换和账号入口沿用现有位置。
- 短链生成组件继续作为首屏中心。
- 首页允许比控制台更清透，但不使用多彩渐变或装饰性浮块。

### 8.2 控制台

- 控制台主区保持工作台感，不添加营销式英雄区。
- 数据行 hover 使用 `surface-strong`，不要使用主色大面积填充。
- 页面级动作继续放在主内容右上角。
- 账号入口继续在桌面侧栏底部，移动端保留顶部头像入口。

### 8.3 认证和初始化页面

- 登录页和初始化页使用同一套主题 token。
- 表单焦点、错误、主按钮和次级说明应与首页、控制台保持一致。
- 不恢复厚重卡片堆叠或大面积装饰渐变。

## 9. 实现边界

实现时优先修改以下位置：

- `web/src/app/vuetify.ts`
- `web/src/app/theme-css.ts`
- `web/src/shared/preferences/PreferenceSwitcher.vue`
- `web/src/widgets/console-shell/ConsoleNavList.vue`
- 与状态标签、输入框和按钮相关的页面局部样式

不应修改：

- v0.0.3 既有页面路由范围。

## 10. 验收标准

- 浅色主题不再使用现有绿色、薄荷或蜂蜜黄作为主视觉。
- 深色主题不再使用现有深墨绿方向，而是蓝黑、钢蓝和冰蓝体系。
- 首页在浅色模式下保持清透、轻量、可读。
- 控制台在深色模式下保持低眩光、层级清晰、可扫描。
- 主按钮、次级按钮、输入焦点、hover、菜单选中和状态标签都有稳定语义。
- 左侧多级导航清晰展示父级展开、子项缩进、当前子项选中和规划中 badge。
- 移动端抽屉导航保留同样的多级结构语义。
- 规划中页面不展示假数据，不开放未实现业务动作。
- 现有 v0.0.2 权限、短链筛选、搜索、用户管理和 E2E 主流程不回退。
- `cd web && pnpm lint`、`pnpm typecheck`、`pnpm test`、`pnpm test:coverage`、`pnpm build` 通过。

## 11. 后续衔接

1. 更新 Vuetify 主题 token。

3. 更新偏好切换菜单的颜色和选中态。
4. 更新控制台多级导航的展开、选中和规划中 badge。
5. 回归首页、控制台、登录页、初始化页和移动端抽屉。
6. 补充或更新对应前端测试。
