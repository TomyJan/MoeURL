# v0.1.0 短链统计分析设计

## 1. 背景与目标

v0.0.4 已持久化成功写出跳转响应的 `redirect_response_sent` 事件，并在短链列表展示总访问量、今日访问量和最近访问时间。控制台的 `/analytics` 入口仍是占位页，事件表也未保存来源、设备和地区维度。

v0.1.0 的目标是将现有事件数据扩展为可用的单短链分析页。用户能够查看自己短链的近 7 天趋势及来源、设备、地区聚合；管理员能够在同一界面查看任意未删除短链的统计。统计仍只表示 MoeURL 已成功写出跳转响应，不表示浏览器已完成跳转或目标页面已加载。

## 2. 范围

### 2.1 必须实现

- 将 `/analytics` 从规划中占位页替换为真实统计分析页。
- 从「我的短链」和「全站短链」列表进入指定短链的分析页。
- 展示总访问量、今日访问量、最近访问时间和连续 7 天访问趋势。
- 展示来源主机名、设备类型和国家/地区的聚合统计。
- 普通用户只能查看自己短链；管理员可查看全站短链。
- 仅在跳转响应成功写出后，异步、尽力持久化本次访问的聚合维度。
- 不持久化原始 IP、完整 User-Agent、完整 Referer 或单次访问明细。

### 2.2 不实现

- 任意日期范围、近 30 天趋势、导出和原始访问明细。
- 浏览器、操作系统、屏幕分辨率或营销归因细分。
- GeoIP 本地数据库、自动下载或更新任务。
- 访问密码、过期时间、二维码、中间页、确认页、用户中心和权限预设。

## 3. 数据与隐私

`short_link_event` 新增以下可空字段，历史事件保持可查询：

| 字段 | 类型 | 生成规则 | 持久化边界 |
| --- | --- | --- | --- |
| `referrer_host` | text | 解析 `Referer` 的小写主机名；缺失、非法或非 HTTP(S) URL 时为空。 | 不保存完整 URL、路径、查询参数或片段。 |
| `device_type` | text | 由 `User-Agent` 归类为 `desktop`、`mobile`、`tablet`、`bot` 或 `other`。 | 不保存原始 User-Agent。 |
| `country_code` | text | 仅读取部署者配置的请求头，规范化为 ISO 3166-1 alpha-2 大写代码；缺失或非法时为空。 | 不保存 IP，也不从 IP 查询地理位置。 |

地区请求头由 `MOEURL_ANALYTICS_COUNTRY_HEADER` 配置；未配置时不采集地区。部署者只能在反向代理会覆盖或移除客户端同名头的环境中启用该配置。该字段仅用于统计展示，不可用于权限、安全或风控判断。

统计页将空维度合并展示为「未知」。维度值按访问量降序排列，最多返回 10 项，其余数据合并为「其他」，避免单个短链造成无界响应体。

## 4. 后端设计

### 4.1 事件写入

`event.Event` 扩展为聚合元数据载体。短链跳转 HTTP handler 在确认响应正文写入成功后，从原始请求构造事件，再调用现有有界 `DBRecorder`。Recorder 保持独立后台任务、500 ms 写入超时和 16 个并发槽位；队列满、字段解析失败和数据库写入失败均只记录结构化日志，绝不改变已发送的跳转响应。

事件表增加 `(short_link_id, created_at)` 索引，并为维度聚合新增以 `short_link_id`、`event_type` 和 `created_at` 为前缀的索引。查询一律限定 `event_type = 'redirect_response_sent'`。

### 4.2 服务与 API

新增两条只读接口，保持既有用户和管理员权限边界：

| 接口 | 权限 | 结果 |
| --- | --- | --- |
| `GET /api/v1/short-link/statistics?id={uuid}` | `short_link:read_own`，且短链归当前用户所有。 | 单条自有短链统计。 |
| `GET /api/v1/admin/short-link/statistics?id={uuid}` | `admin:access` 与 `short_link:read_all`。 | 任意未删除短链统计。 |

`id` 缺失或不是 UUID 返回既有 `100001` 业务错误；短链不存在、已软删除或不属于当前用户返回 `200104`；权限不足返回 `120001`。基础设施失败仍返回 HTTP `500`。

成功响应结构为：

```json
{
  "shortLink": {
    "id": "...",
    "url": "https://example.com/abc123",
    "slug": "abc123",
    "targetUrl": "https://target.example/",
    "status": "active"
  },
  "stats": {
    "visitCount": 42,
    "todayVisitCount": 4,
    "lastVisitedAt": "2026-07-17T10:00:00Z",
    "trend": [{ "date": "2026-07-11", "visitCount": 0 }],
    "referrers": [{ "value": "search.example", "visitCount": 12 }],
    "devices": [{ "value": "mobile", "visitCount": 20 }],
    "countries": [{ "value": "CN", "visitCount": 8 }]
  }
}
```

趋势固定为服务端当前日期及此前 6 个日历日，按数据库时区聚合，响应总是返回 7 个升序日期桶。概览字段与 v0.0.4 的口径保持一致。

## 5. 控制台设计

保留 `/analytics` 路径，使用 `shortLinkId` 查询参数指定目标短链。没有参数时显示当前用户可见短链的选择状态；无权或不存在时显示 API 的业务错误状态。列表每行增加带 tooltip 的分析图标，跳转到对应 URL。

页面由以下区域组成：

- 页面标题和所选短链 URL，提供返回短链列表的动作。
- 3 个紧凑指标：总访问量、今日访问量、最近访问时间。
- 近 7 天折线图。使用 `chart.js`，空桶也要显示日期与零值。
- 来源、设备、地区三个并列聚合表。小屏幕依次纵向排列，不产生横向滚动。

所有新增展示文案必须同时提供简体中文和英语翻译。分析导航项移除「规划中」标识；其他未实现入口保持原状。

## 6. 验收与测试

- Goose migration 可升级和回滚，SQLC 生成代码与查询同步。
- 数据库集成测试覆盖：历史空字段、7 个连续趋势桶、来源/设备/地区聚合、维度截断与「其他」、软删除排除。
- Recorder 与 redirect handler 测试覆盖：成功响应采集、失败响应不采集、字段规范化、槽位满和写入失败不阻断。
- Service、handler 和 router 测试覆盖所有权、管理员访问、非法 ID、业务错误和 HTTP `500`。
- Vitest 覆盖 API 映射、分析页面的加载、空态、错误态、图表数据和列表入口。
- Playwright 在 CI 中运行登录、创建短链、访问短链、进入统计页的主流程；不得再仅以本机镜像拉取问题替代 E2E 验收。
- `go test ./...`、`go vet ./...`、后端 100% 覆盖率门禁、`pnpm lint`、`pnpm typecheck`、`pnpm test`、`pnpm test:coverage`、`pnpm build` 和 E2E 全部通过。
