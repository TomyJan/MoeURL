# v0.0.3 UI/UX 精细化重设计

## 1. 背景

v0.0.1 和 v0.0.2 已完成 MoeURL 的基础可用闭环、管理员用户维护、短链筛选和部署验证。当前 UI 仍偏临时实现，页面结构、视觉风格和控制台体验尚未形成稳定设计系统。

v0.0.3 的目标是完成第一轮 UI/UX 精细化重设计，让 MoeURL 从「能用」推进到「像一个完整产品」。本阶段优先处理首页、控制台框架、主题配色、大圆角视觉系统和短链生成入口，不引入统计、中间页跳转、访问密码等新业务能力。

## 2. 设计目标

- 首页首屏聚焦短链快速生成，导航融入页面视觉，不做独立背景的传统导航栏。
- 控制台采用左侧导航布局，移动端改为汉堡菜单和抽屉导航。
- 建立新的浅色和深色主题基准，替换当前临时配色。
- 圆角显著增大，形成柔和、现代、轻量的产品气质。
- 短链生成组件抽成可复用能力，同时服务首页和控制台。
- 保持 MoeURL 自托管、轻量、可控的定位，不做重营销页或复杂 SaaS 后台。

## 3. 非目标

- 不在 v0.0.3 实现统计能力。
- 不在 v0.0.3 实现访问密码、过期时间、中间页跳转和二维码。
- 不在 v0.0.3 实现完整用户中心仪表盘。
- 不在 v0.0.3 实现主题后台配置。
- 不重写后端业务模型和 API 契约，除非 UI 复用组件需要极小范围的前端适配。

## 4. 视觉方向

### 4.1 浅色主题

浅色主题采用「清透产品感」方向。

建议 token：

| 用途 | 建议值 | 说明 |
| --- | --- | --- |
| 页面背景 | `#eff6f2` | 浅薄荷底色，替代纯白后台感。 |
| 主色 | `#0f4f48` | 深青绿，用于主按钮、Logo 和选中态。 |
| 强调色 | `#f0a94f` | 蜂蜜黄，只用于少量提示、图形和高亮。 |
| 主文字 | `#14302b` | 低饱和深绿黑，避免纯黑生硬。 |
| 次级文字 | `#5d766d` | 用于说明、辅助信息和空状态。 |
| 表面色 | `#ffffff` | 卡片、表格、浮窗主体。 |
| 柔和表面 | `#f6fbf8` | 控制台主内容背景。 |
| 边框 | `rgba(15, 79, 72, 0.10)` | 轻分隔，不使用重线框。 |

### 4.2 深色主题

深色主题采用「深墨绿清透」方向，对应浅色主题的夜间形态。

建议 token：

| 用途 | 建议值 | 说明 |
| --- | --- | --- |
| 页面背景 | `#10211e` | 深墨绿，低眩光。 |
| 主内容背景 | `#111816` | 控制台深色主区。 |
| 表面色 | `#17231f` | 卡片、列表和浮窗主体。 |
| 强表面色 | `#17332e` | 侧栏选中、抽屉和强调容器。 |
| 主色 | `#65d6b1` | 薄荷绿，用于主按钮和关键状态。 |
| 暖强调色 | `#ecb65c` | 少量继承浅色主题的温度，不作为大面积按钮色。 |
| 主文字 | `#effdf6` | 深色主文字。 |
| 次级文字 | `#9eb6ad` | 深色辅助文字。 |
| 边框 | `rgba(255, 255, 255, 0.08)` | 深色轻分隔。 |

### 4.3 圆角规则

v0.0.3 应明显提高圆角，但仍保持层级差异。

| 元素 | 圆角建议 |
| --- | --- |
| 页面级大容器 | 32px-40px |
| 控制台面板、表格容器、浮窗 | 28px-38px |
| 卡片、统计块、抽屉分组 | 24px-30px |
| 输入框、按钮、筛选 chip | 999px 或 18px-24px |
| Logo 方块 | 14px-18px |

按钮和输入框可使用胶囊形态。表格本身不做硬边框，放在大圆角容器中，通过轻分隔线组织信息。

## 5. 首页设计

### 5.1 顶部导航

首页顶部导航不使用独立背景色，不做传统 `v-app-bar` 式悬浮栏。导航应融入首页背景。

布局：

- 左侧：MoeURL Logo 和站点名称。
- 右侧未登录：登录按钮。
- 右侧已登录：头像 + 昵称入口。
- 点击头像昵称进入控制台，而不是展开复杂菜单。
- 语言和主题切换可作为小型图标按钮放在右侧，但不得压过登录或账号入口。

### 5.2 首屏结构

首页首屏应占满视口主要空间，核心内容居中。

首屏内容：

- 一句产品主标题，表达短链生成的价值。
- 一句简短说明，不写冗长营销文案。
- 短链快速生成组件。
- 无权限时保留表单结构，但按钮禁用并提示登录或联系管理员。

首屏下方应露出下一段内容的视觉提示，告诉用户可以继续向下滚动。

### 5.3 下翻内容

首页可以向下滚动，用于承载轻量介绍。

建议区块：

- 权限可控：说明能力由权限控制。
- 自托管友好：说明部署和数据控制。
- 快速管理：说明可以在控制台管理短链。
- 现代体验：说明主题、多语言和 PWA 基础能力。

这些介绍不应喧宾夺主。首页首要任务仍是生成短链。

## 6. 控制台设计

### 6.1 桌面布局

控制台采用左侧导航 + 主内容区。

左侧导航包含：

- 顶部品牌区：Logo、站点名称和简短上下文。
- 全局主动作：「新建短链」按钮。
- 工作区分组：
  - 概览。
  - 我的短链。
  - 统计分析（规划中占位页，不展示假统计）。
- 管理分组：
  - 全站短链。
  - 用户管理（父级默认折叠，展开后包含用户、用户组与权限）。
  - 系统设置（规划中占位页，不开放表单）。
- 底部账号卡：
  - 头像。
  - 昵称。
  - 个人设置入口。
  - 退出登录入口。

主内容顶部不放常驻账号菜单。右上角只放页面级动作，例如「新建短链」「导出」「筛选」。

### 6.2 账号入口决策

桌面控制台中，头像菜单放到左侧导航底部，而不是主区右上角。

原因：

- 账号入口属于全局身份和设置，不是页面级操作。
- 主区右上角应留给当前页面动作。
- 左侧底部账号卡更稳定，也能减少顶部拥挤。

移动端例外。移动端侧栏默认收起，因此顶部右侧保留头像按钮，点击打开账号菜单。

### 6.3 移动布局

移动端控制台采用：

- 顶部：汉堡按钮、页面标题、头像按钮。
- 汉堡按钮打开左侧抽屉导航。
- 头像按钮打开账号菜单。
- 主内容纵向堆叠。
- 「新建短链」可放在页面顶部主按钮，或作为底部固定动作按钮。
- 生成浮窗在移动端表现为底部抽屉。

### 6.4 控制台页面基线

v0.0.3 不要求补齐所有后台功能，但应建立页面框架。

优先覆盖：

- 控制台外壳。
- 我的短链页。
- 全站短链页。
- 用户管理页。
- 创建用户页。

这些页面应迁移到统一控制台框架中，避免每页独立铺 `v-container`。

概览、统计分析、用户组与权限、系统设置可以作为规划中占位页面出现。占位页只用于稳定信息架构，必须清楚说明能力尚未实现，不展示假数据，也不提供不可用表单。

## 7. 短链生成组件

### 7.1 组件定位

短链生成应抽象为可复用组件，避免首页和控制台各写一套逻辑。

建议组件能力：

```text
ShortLinkCreatePanel
  variant: inline | modal | bottom-sheet
  mode: compact | full
```

### 7.2 使用场景

首页：

- 使用 `inline` 外壳。
- 使用 `full` 或接近完整的首屏展示。
- 作为页面视觉中心。

控制台桌面端：

- 左侧导航和列表页右上角提供「新建短链」按钮。
- 点击后打开 `modal` 外壳。
- modal 内使用同一套生成表单。

控制台移动端：

- 点击「新建短链」后打开 `bottom-sheet` 外壳。
- 表单内容与桌面 modal 保持一致。

### 7.3 表单结构

v0.0.3 仍以当前业务能力为准。

必须展示：

- 目标 URL 输入框。
- 生成按钮。
- 成功后的短链展示。
- 复制短链操作。
- 权限不足提示。
- 错误状态。

可以预留但不实现完整流程：

- 短码。
- 域名。
- 状态。
- 高级选项折叠区。

预留项不得造成用户误以为 v0.0.3 已支持后续能力。

## 8. 组件和布局规则

### 8.1 导航

- 首页导航视觉融入背景，不使用独立导航背景。
- 控制台导航常驻左侧。
- 移动端使用汉堡菜单和抽屉。
- 管理入口根据权限展示。

### 8.2 页面容器

- 首页允许使用更强的视觉布局。
- 控制台主区保持工作台感，不做营销式大标题。
- 页面标题、说明、动作按钮应形成稳定结构。
- 列表页优先清晰和可扫描，不追求装饰。

### 8.3 表格和列表

- 表格放在大圆角 surface 中。
- 表头使用低对比辅助文字。
- 行分隔线保持轻量。
- 状态使用 pill 样式。
- 行操作优先使用图标或短文本按钮，避免过宽。

### 8.4 空状态和错误状态

- 空状态应保留下一步动作，例如「创建短链」。
- 错误状态应提供重试或返回入口。
- 权限不足应说明原因，避免只显示空白页。

## 9. 与现有技术栈的关系

v0.0.3 继续使用 Vue 3、Vuetify 4、TypeScript、Pinia、TanStack Query 和 vue-i18n。

实现时建议：

- 在 Vuetify theme 中定义新的浅色和深色主题 token。
- 继续使用 Vuetify 组件，但通过自定义 class 和主题 token 去除默认后台模板感。
- 抽出控制台布局组件，例如 `ConsoleShell`。
- 抽出首页顶部组件，例如 `HomeHeader`。
- 抽出短链生成组件，例如 `ShortLinkCreatePanel`。
- 保留现有 API 和 TanStack Query 数据流。

## 10. 验收标准

- 首页首屏在桌面和移动端都以短链快速生成作为视觉中心。
- 首页顶部导航没有独立背景，左侧品牌、右侧登录或头像昵称入口清晰。
- 已登录用户可从首页头像昵称进入控制台。
- 控制台桌面端使用左侧导航。
- 控制台移动端使用汉堡菜单和抽屉导航。
- 桌面控制台账号入口位于左侧导航底部。
- 移动端账号入口位于顶部右侧。
- 控制台中可以通过「新建短链」按钮打开生成浮窗。
- 首页和控制台复用同一套短链生成逻辑。
- 浅色主题符合「清透产品感」方向。
- 深色主题符合「深墨绿清透」方向。
- 主要容器、卡片、浮窗和按钮采用新的大圆角规则。
- 我的短链、全站短链、用户管理和创建用户页面纳入统一控制台框架。
- 现有 v0.0.2 功能和权限行为不回退。
- 前端 lint、typecheck、test、coverage 和 build 通过。

## 11. 实现决策

- v0.0.3 可以展示「统计分析」和「系统设置」等规划中占位入口，但页面内容必须明确标记为规划中。
- 后续能力不使用禁用导航项伪装已实现功能，避免用户误以为当前版本已经支持完整流程。
- 首页下翻介绍区使用产品 UI 片段、图形化信息块和简短文字，不引入外部图库依赖。
- 控制台概览页如果暂不具备真实统计数据，应优先展示当前可用的链接、用户和权限状态，不伪造统计能力。

## 12. 后续衔接

1. 新主题 token 和全局视觉基线。
2. 可复用短链生成组件。
3. 首页重设计。
4. 控制台 Shell 和导航。
5. 我的短链、全站短链、用户管理和创建用户页面迁移。
