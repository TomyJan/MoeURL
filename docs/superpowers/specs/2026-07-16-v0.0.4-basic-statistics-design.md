# v0.0.4 基础统计前置设计

## 背景

v0.0.3 已完成首页、控制台和短链生成体验重设计，但短链列表仍只能展示短链自身状态，无法反映访问表现。路线图中的 v0.1.0 计划包含基础统计、维度统计、访问密码、过期时间和二维码等多项能力。直接进入 v0.1.0 范围过大，v0.0.4 先实现基础统计数据闭环。

## 目标

v0.0.4 建立短链访问事件持久化，并在既有短链列表中展示基础统计。

必须交付：

- 成功写出跳转响应事件入库。
- 我的短链和全站短链列表展示总访问量、今日访问量和最近访问时间。
- 保持现有权限边界和列表交互不回退。

## 非目标

v0.0.4 不实现：

- 独立统计分析页。
- 短链详情页。
- 趋势图表。
- 访问明细。
- 来源、设备、浏览器、系统和地区维度。
- 统计导出。
- IP、User-Agent 和 Referer 持久化。
- 访问密码、过期时间、二维码和新的跳转模式。

## 统计口径

访问量只统计 `redirect_response_sent` 事件。该事件表示 MoeURL 已成功写出跳转响应，不代表目标站点实际加载成功。

以下情况不计入访问量：

- 短链不存在。
- 短链已软删除。
- 短链已禁用。
- 后续访问条件拦截。

## 数据模型

新增 `short_link_event` 表：

- `id uuid primary key`
- `short_link_id uuid not null references short_link(id)`
- `event_type text not null`
- `created_at timestamptz not null`

索引：

- `short_link_event_short_link_id_idx`
- `short_link_event_type_created_at_idx`
- `short_link_event_short_link_created_at_idx`

本版本不保存 IP、User-Agent、Referer 或地区信息。

## 后端设计

`internal/event` 保留统一事件接口，并新增数据库 recorder。`RedirectService` 继续通过 `event.Recorder` 记录事件，避免跳转流程直接依赖 SQL 细节。

短链列表 SQL 使用聚合子查询返回：

- `visit_count`
- `today_visit_count`
- `last_visited_at`

`today_visit_count` 以数据库当前日期为准，统计 `created_at >= current_date` 的成功响应事件。

事件记录失败不阻断跳转。recorder 会对非法短链 ID 做同步校验，数据库写入失败或后台写入超时可直接丢弃统计事件，避免拖慢跳转请求路径。

## 前端设计

短链模型增加：

```typescript
stats: {
  visitCount: number
  todayVisitCount: number
  lastVisitedAt: string | null
}
```

`ConsoleLinkList` 在每条短链行中展示：

- 总访问。
- 今日访问。
- 最近访问。

没有最近访问时间时展示「暂无访问」。移动端沿用现有数据行结构，统计信息作为紧凑元信息展示，不新增嵌套卡片。

## 测试策略

后端：

- migration 测试覆盖新表。
- recorder 测试覆盖事件写入。
- redirect 测试覆盖成功写出跳转响应后写事件、禁用短链不写成功访问。
- service/handler 测试覆盖列表统计字段。

前端：

- API 测试覆盖统计字段。
- 页面或列表组件测试覆盖统计展示和无访问占位。

## 自检结论

- 范围聚焦在基础统计闭环，未混入完整统计页和维度统计。
- 统计口径与现有事件规格一致。
- 权限边界沿用既有短链列表权限，不新增权限模型。
- 数据模型保留后续扩展空间，但不提前持久化敏感访问明细。
